Convert CP866 to UTF-8 on Mac

18 Jan 2023 AlexS
bash macOS

Old text files — especially those originating from DOS or early Windows systems — often use the CP866 character encoding. Opening these files in modern editors results in garbled, unreadable characters. Converting them to UTF-8 is essential for proper display and further processing.

macOS includes the iconv utility out of the box, making character encoding conversions a one-line operation.

Why Convert CP866?

  • Readability — garbled text becomes properly rendered Cyrillic characters
  • Compatibility — UTF-8 is the universal standard for modern applications and editors
  • Data integrity — prevents corruption when importing legacy data into databases or APIs
  • Zero extra tools — iconv ships with macOS, no Homebrew install needed

The Solution

A single iconv command converts an entire file from CP866 to UTF-8:

iconv -f cp866 -t utf-8 < file.txt > new_file.txt

The converted output is written to new_file.txt, leaving the original untouched.

Understanding the Flags

FlagMeaning
-f cp866Specifies the from encoding (source format)
-t utf-8Specifies the to encoding (target format)
< file.txtReads input from the original CP866-encoded file
> new_file.txtRedirects converted output to a new UTF-8 file

How It Works

  1. iconv reads the raw bytes from file.txt
  2. It interprets those bytes as CP866 characters
  3. Each character is mapped to its UTF-8 equivalent
  4. The result is streamed into the new file via shell redirection

Verify the conversion with the file command:

file new_file.txt
# Output: new_file.txt: Unicode text, UTF-8

Practical Examples

Convert Multiple Files at Once

for f in *.txt; do
    iconv -f cp866 -t utf-8 < "$f" > "utf_${f}"
done

In-Place Conversion (Overwrite Original)

iconv -f cp866 -t utf-8 < file.txt > tmp.txt && mv tmp.txt file.txt

Handle Conversion Errors Gracefully

Some characters may not map cleanly. Use //TRANSLIT for approximate replacements or //IGNORE to skip unmappable characters:

# Skip problematic characters silently
iconv -f cp866 -t utf-8//IGNORE < file.txt > new_file.txt

# Approximate with closest available character
iconv -f cp866 -t utf-8//TRANSLIT < file.txt > new_file.txt

Customization Tips

  • Different source encoding — replace cp866 with iso-8859-1, windows-1251, koi8-r, etc. (run iconv -l to list all supported encodings)
  • Recursive directory conversion — combine with find to batch-process entire folder trees
  • Mac Terminal encoding — ensure your terminal is set to UTF-8 in Terminal.app → Preferences → Profiles → Text

A one-line command that rescues decades of legacy text files. Convert once, read forever.
