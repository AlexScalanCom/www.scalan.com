List Directory Sizes in Linux with du

19 Dec 2020 AlexS
bash linux DevOps

Understanding where your disk space goes is one of the first troubleshooting steps when storage runs low. The du (disk usage) command provides an instant breakdown of directory sizes without needing any third-party tools. Combined with a few flags, it gives you a clean, human-readable overview of what's consuming space.

This guide shows how to list directory sizes for visible and hidden folders alike.

Why Use du for Disk Analysis?

  • Instant visibility — see which directories consume the most space at a glance
  • No installation needed — du is part of GNU coreutils, available on every Linux distribution
  • Human-readable output — sizes displayed in KB, MB, GB instead of raw bytes
  • Quick decision making — identify bloat directories to clean up or move to external storage

The Solution

List Visible Directory Sizes

Get a summary of all non-hidden directories in the current path:

du -sh *

Include Hidden Directories

To capture dotfiles and hidden folders as well:

sudo du -sh $(ls -A)

The sudo prefix ensures you can read directories restricted to root.

Understanding the Flags

FlagPurpose
duDisk usage utility that estimates space consumed by files and directories
-sSummarize — show only the total for each argument, not every subdirectory
-hHuman-readable — display sizes as K, M, G instead of byte counts
*Shell glob that expands to all non-hidden entries in the current directory
$(ls -A)Command substitution that lists all entries including hidden ones (excludes . and ..)

Note: Using sudo may be necessary when listing directories owned by root or other users that your account cannot traverse.

How It Works

  1. Glob expansion — * or $(ls -A) expands to a list of directory names in the current path
  2. du traversal — for each directory, du walks the subtree and sums file block sizes
  3. Summarize — the -s flag collapses each subtree into a single total line
  4. Human formatting — the -h flag converts raw bytes into KB, MB, or GB with appropriate suffixes

Practical Examples

Find Largest Directories in /var

sudo du -sh /var/* | sort -h | tail -10

Pipes du output through sort -h (human-aware sorting) and shows the top 10 largest directories.

Check Home Directory Breakdown

du -sh ~/* ~/.* | grep -v '\.\.\?$' | sort -h | tail -10

Lists both visible and hidden home directories, filters out . and .., then ranks by size.

Deep Dive into a Specific Folder

Drop the -s flag to see recursive subdirectory sizes:

du -h --max-depth=2 /opt/myapp | sort -h

Shows sizes two levels deep within /opt/myapp.

Customization Tips

  • Sort by size — pipe through sort -h to order results from smallest to largest
  • Limit depth — use --max-depth=N instead of -s for multi-level breakdowns
  • Exclude patterns — add --exclude='*.log' or --exclude=node_modules to skip known bloat
  • One-liner top-N — combine with sort -rh | head -5 to see the 5 biggest directories instantly

A single command that reveals where your disk space disappears. Run it, sort it, clean it up.
