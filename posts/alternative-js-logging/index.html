Alternative JS Logging

29 Jan 2025 AlexS
javascript DevOps

Problem: Debugging client-side JavaScript on embedded devices, kiosks, or restricted environments where you cannot open DevTools or access the browser console.

Solution Overview

The approach is simple yet effective: send log messages from the device via an asynchronous HTTP request to a lightweight PHP endpoint on your web server. The PHP script captures the caller's IP address and appends a timestamped entry to a centralized log.txt file — giving you a remotely accessible debug trail.

Step 1 — Server-Side Logger (doLog.php)

Deploy the following PHP script on any Apache/Nginx web server with write permissions for the log file:

<?php
if (isset($_GET["q"])) {
    // Determine client IP address
    $ip = "-";
    if (!empty($_SERVER['HTTP_CLIENT_IP'])) {
        $ip = $_SERVER['HTTP_CLIENT_IP'];
    } elseif (!empty($_SERVER['HTTP_X_FORWARDED_FOR'])) {
        $ip = $_SERVER['HTTP_X_FORWARDED_FOR'];
    } else {
        $ip = $_SERVER['REMOTE_ADDR'];
    }

    // Append timestamped log entry
    $fp = fopen('log.txt', 'a');
    fwrite($fp, date("Y-m-d H:i:s", time()) . " [" . $ip . "] " . $_GET["q"] . "\n");
    fclose($fp);

    echo "File appended successfully";
} else {
    echo "Usage: doLog.php?q=your_log_message";
}
?>

What it does:

  • Extracts the client IP (supporting direct, proxied, and load-balanced connections).
  • Prepends a YYYY-MM-DD HH:MM:SS timestamp to every entry.
  • Appends the message to log.txt in a human-readable format.

Step 2 — Client-Side Logger (JavaScript)

Add this helper function to any JavaScript file running on the restricted device:

function log(message) {
  const xhr = new XMLHttpRequest();
  xhr.open(
    "GET",
    "https://your-server.com/doLog.php?q=" + encodeURIComponent(message),
  );
  xhr.send();
}

Usage example:

const val = 42;
log("Processing complete — val=" + val);

Tip: Using encodeURIComponent() ensures that special characters, spaces, and URL-sensitive symbols in the message don't break the request.

Accessing the Logs

Open https://your-server.com/log.txt in any browser to view all collected entries in real time. Each line follows this format:

2025-01-29 14:32:07 [192.168.1.45] Processing complete — val=42

When to Use This Approach

ScenarioFit
Kiosk / embedded device debugging✅ Ideal
Remote IoT client logging✅ Ideal
Production browser apps with console access⚠️ Overkill — use standard console + log aggregators
High-throughput logging⚠️ Consider a dedicated service (ELK, Datadog) instead

Security Notes

  • Validate input — In production, sanitize $_GET["q"] to prevent log injection attacks.
  • Restrict access — Protect the endpoint with authentication or IP whitelisting if logs contain sensitive data.
  • Rotate logs — Implement log rotation (logrotate) to prevent unbounded file growth on disk.

This minimal, zero-dependency pattern gives you immediate visibility into client-side execution on devices where traditional debugging tools simply aren't available.
