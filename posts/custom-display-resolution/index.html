Set Custom Display Resolution on Linux

22 Mar 2023 AlexS
bash linux

Small monitors, odd aspect ratios, and mismatched EDID data can leave your Linux desktop stuck at an uncomfortable resolution. The xrandr utility gives you full control over display modes — letting you set custom resolutions and panning areas on the fly.

This guide shows how to check your current display configuration, apply a custom resolution, and restore defaults.

Why Custom Resolution?

  • Small monitors — fit content properly on compact or embedded displays
  • Odd aspect ratios — work around EDID limitations that don't advertise optimal modes
  • Panning support — enable virtual desktop areas larger than the physical screen for multitasking
  • Zero reboot needed — changes apply instantly via command line

Check Current Display

First, identify your active output and available modes:

xrandr -q

Look for the connected output (e.g., HDMI-1, DP-0, VIRTUAL1) and its current resolution in the output:

Screen 0: minimum 320 x 200, current 1920 x 1080, maximum 7680 x 7680
HDMI-1 connected primary 1920x1080+0+0 ...
   1920x1080     60.00*+
   1360x768      59.89

The * marks the active mode; + indicates preferred EDID modes.

Set Custom Resolution

Apply a custom mode with optional panning to create a larger virtual desktop:

xrandr --output HDMI-1 --mode 1360x768 --panning 4096x2048

Replace HDMI-1 with your actual output name. The --panning flag defines the maximum virtual area, allowing you to drag windows beyond the physical screen edges.

Understanding the Flags

FlagPurpose
--output HDMI-1Targets the specific display connector (use xrandr -q to find yours)
--mode 1360x768Sets the active resolution mode
--panning 4096x2048Defines the virtual desktop area for window panning

Restore Default Resolution

Quickly revert to a standard resolution:

xrandr -s 1360x768

Or re-select any mode listed by xrandr -q:

xrandr --output HDMI-1 --mode 1920x1080

How It Works

  1. Query — xrandr -q reads the X server's display configuration and lists available modes
  2. Apply — --mode switches to the specified resolution; --panning extends the virtual area
  3. Revert — -s shortcut quickly resets to any previously defined mode without re-specifying the output

Customization Tips

  • Create a new mode — use cvt 1360 768 to generate modeline parameters, then add it with xrandr --newmode and xrandr --addmode HDMI-1 <name>
  • Make it persistent — add the xrandr command to ~/.xinitrc, ~/.config/lxsession/LXDE-pi/autostart, or a systemd user service
  • Rotate display — append --rotate normal|left|right|inverted for landscape or portrait orientations
  • Multi-monitor setup — chain multiple --output blocks to configure each screen independently

A few commands that give you pixel-perfect control over any display. Resize, pan, and move on.
