Map JPA Entity Fields to Database Column Names

19 Aug 2022 AlexS
java springboot

JPA entities map Java field names to database column names via @Column annotations. Sometimes you need these mappings at runtime — for dynamic query builders, audit logging, or debugging ORM behavior. Reflection makes it possible to inspect any entity class and extract a complete column-to-field name map without hardcoding anything.

This guide shows two utility methods: one that maps column → field names, and another that maps field → column names.

Why Extract Column Mappings?

  • Dynamic queries — build WHERE clauses using database column names while referencing Java fields programmatically
  • Audit trails — log which entity fields changed by mapping database triggers back to Java property names
  • Debugging ORM mismatches — quickly spot discrepancies between expected and actual column mappings
  • Generic utilities — write reusable data comparison or diff tools that work across any JPA entity

The Solution

Column → Field Names

Returns a map where keys are database column names and values are Java field names:

public static Map<String, String> entityColumnsFieldNames(Class<?> entityClass) {
    Map<String, String> result = new HashMap<>();
    Field[] fields = entityClass.getDeclaredFields();

    for (Field field : fields) {
        Column col = field.getAnnotation(Column.class);
        if (col != null) {
            result.put(col.name(), field.getName());
        }
    }
    return result;
}

Field → Column Names

Returns a map where keys are Java field names and values are database column names:

public static Map<String, String> entityFieldsColumnNames(Class<?> entityClass) {
    Map<String, String> result = new HashMap<>();
    Field[] fields = entityClass.getDeclaredFields();

    for (Field field : fields) {
        Column col = field.getAnnotation(Column.class);
        if (col != null) {
            result.put(field.getName(), col.name());
        }
    }
    return result;
}

Understanding the Reflection

StepWhat It Does
entityClass.getDeclaredFields()Retrieves all fields (private, protected, public) declared in the entity class
field.getAnnotation(Column.class)Checks if the field has a JPA @Column annotation
col.name()Extracts the database column name from the annotation attribute
field.getName()Gets the Java field name as a string
result.put(key, value)Stores the mapping in the result map

Note: Fields without explicit @Column annotations are skipped. If a field uses the default naming strategy (field name = column name), it won't appear in the map unless explicitly annotated.

How It Works

  1. Reflection scan — getDeclaredFields() collects all fields from the entity class
  2. Annotation check — each field is tested for a @Column annotation
  3. Name extraction — the annotation's name() attribute and the field's Java name are both captured
  4. Map population — key-value pairs are added to the returned Map<String, String>

Practical Examples

Inspect an Order Entity

// Given this entity:
@Entity
class Order {
    @Column(name = "order_id")
    private Long id;

    @Column(name = "customer_email")
    private String email;

    @Column(name = "total_amount")
    private BigDecimal total;
}

// Column → Field map:
Map<String, String> colToField = entityColumnsFieldNames(Order.class);
// {"order_id"="id", "customer_email"="email", "total_amount"="total"}

// Field → Column map:
Map<String, String> fieldToCol = entityFieldsColumnNames(Order.class);
// {"id"="order_id", "email"="customer_email", "total"="total_amount"}

Build a Dynamic WHERE Clause

String fieldName = "email";
Map<String, String> mapping = entityFieldsColumnNames(User.class);
String columnName = mapping.get(fieldName);
// Produces: "WHERE customer_email = ?"

Customization Tips

  • Include inherited fields — walk the class hierarchy with entityClass.getSuperclass().getDeclaredFields() to capture parent entity mappings
  • Handle default naming — add a fallback that maps unannotated fields using the JPA default naming strategy (field.getName())
  • Generic type safety — wrap in a @UtilityClass (Lombok) or a sealed helper class to prevent instantiation
  • Cache the result — store mappings in a ConcurrentHashMap<Class<?>, Map<String, String>> since entity structure doesn't change at runtime
