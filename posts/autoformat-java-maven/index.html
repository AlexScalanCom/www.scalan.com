Auto-Format Java Sources with Maven

07 Sep 2024 AlexS
java maven

Consistent code formatting is a cornerstone of maintainable Java projects. Manual formatting leads to inconsistencies, noisy diffs, and wasted time in code reviews. The Spotless Maven plugin solves all of this by automatically applying a standardized style to your source files on every build.

This guide walks through integrating auto-formatting into a Maven project using Google's Java Style Guide.

Why Auto-Format?

  • Consistency — every file adheres to the same formatting rules without manual intervention
  • Cleaner diffs — code reviews focus on logic, not whitespace debates
  • Automatic enforcement — formatting runs as part of the standard build lifecycle
  • Onboarding friendly — new team members get proper formatting on their first mvn package

Setup

Add the Spotless plugin configuration to the <build><plugins> section of your pom.xml:

<build>
    <plugins>
        <!-- ... other plugins ... -->

        <!-- Spotless -->
        <plugin>
            <groupId>com.diffplug.spotless</groupId>
            <artifactId>spotless-maven-plugin</artifactId>
            <version>2.43.0</version>
            <configuration>
                <java>
                    <includes>
                        <include>src/main/java/**/*.java</include>
                        <include>src/test/java/**/*.java</include>
                    </includes>
                    <googleJavaFormat>
                        <version>1.23.0</version>
                        <style>GOOGLE</style>
                    </googleJavaFormat>
                </java>
            </configuration>
            <executions>
                <execution>
                    <goals>
                        <goal>apply</goal>
                    </goals>
                    <phase>package</phase>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

Configuration Breakdown

ElementWhat It Does
<includes>Targets all .java files under src/main/java and src/test/java
<googleJavaFormat>Applies Google Java Style Guide
<version>1.23.0</version>Specifies the Google Java Format library version
<style>GOOGLE</style>Uses the standard Google style (alternatives: AOSP, EARLY_JAVA_BLOCKS)
<phase>package</phase>Runs formatting during the package lifecycle phase
<goal>apply</goal>Actually rewrites the files with proper formatting

How It Works

With the plugin configured, formatting triggers automatically whenever you package your project:

mvn package

During the package phase, Spotless reformats all matched Java files before compilation. You can also run formatting as a standalone command:

# Format all sources
mvn spotless:apply

# Check for unformatted files (fails if changes needed)
mvn spotless:check

Customization Tips

  • Different style guide — change <style>GOOGLE</style> to AOSP or use <palantirJavaFormat> instead
  • Add more rules — include <removeUnusedImports/>, <trimTrailingWhitespace/>, or <endWithNewline/> inside <java>
  • CI enforcement — run mvn spotless:check in your pipeline to reject improperly formatted code
  • IDE integration — configure your IDE (IntelliJ, Eclipse) to run spotless:apply on file save for instant formatting

Formatted code, zero arguments. Plug Spotless into your Maven build and let the machine handle style.
