Copy Files from Docker Container to Host

01 Sep 2021 AlexS
bash linux docker

Docker containers are ephemeral by design — when a container stops, its filesystem disappears unless you explicitly preserve it. But what if you need to extract a log file, configuration, or generated artifact from a running container without relying on volume mounts? The docker cp command makes this a one-liner.

This guide shows how to copy files and directories between a Docker container and your host machine.

Why Use docker cp?

  • Debugging — grab log files or config dumps from a live container for inspection
  • Data recovery — rescue generated artifacts before a container is destroyed
  • No volume planning needed — works even when you forgot to mount a volume upfront
  • Bidirectional — copy files into containers just as easily as extracting them

Copy File from Container to Host

The basic syntax extracts a file from a running (or stopped) container:

docker cp <container>:<path>/<filename> <local-path>

For example, copy an Nginx log file to your current directory:

docker cp nginx:/var/log/nginx/mylog.log .

Copy Directory from Container to Host

Entire directories work the same way:

docker cp nginx:/etc/nginx/conf.d ./nginx-config/

All files inside conf.d/ will be recursively copied into your local nginx-config/ folder.

Copy File from Host to Container

Reverse the direction to push files into a running container:

docker cp myapp.properties nginx:/app/config/myapp.properties

This is useful for hot-patching configurations without rebuilding the image.

Understanding the Syntax

ComponentPurpose
docker cpThe Docker CLI command for file copying
<container>Container name or ID (e.g., nginx, a1b2c3d4e5f6)
<path>/<filename>Full path inside the container's filesystem
<local-path>Destination path on your host machine (. means current directory)

Tip: Container doesn't need to be running — docker cp works on stopped containers too.

How It Works

  1. Identify the container name or ID with docker ps -a
  2. Locate the file path inside the container (use docker exec <container> ls /path to explore)
  3. Copy using docker cp — Docker streams the file through the daemon to your host
  4. Verify the extracted file exists at the destination

Practical Examples

Copy Multiple Files with a Pattern

docker cp nginx:/var/log/nginx/*.log ./logs/

Extract and Inspect Container Config

docker cp mydb:/etc/postgresql/postgresql.conf .
cat postgresql.conf | grep max_connections

Quick Debug: Dump Environment Variables

docker cp myapp:/proc/1/environ ./app-environ

Customization Tips

  • Find container ID — run docker ps for running containers or docker ps -a to include stopped ones
  • Explore container filesystem — use docker exec <container> find / -name "*.log" to locate files
  • Preserve permissions — docker cp preserves file ownership; add --archive flag in newer Docker versions for full metadata retention
  • Alternative: volumes — for ongoing access, prefer bind mounts (-v ./data:/app/data) over repeated docker cp commands

A simple command that bridges the container boundary. Extract what you need, debug faster, lose nothing.
