Clean Up Old Files in Temp Directories with find

15 Sep 2023 AlexS
bash linux DevOps

Temporary directories accumulate old files over time — cached data, expired sessions, leftover build artifacts. Without regular cleanup, /tmp and custom temp folders grow until they consume all available disk space. The find command combined with -mtime provides a simple, efficient way to enforce retention policies without extra dependencies.

This guide shows how to delete files older than 30, 60, or 90 days using tiered temp directories — and schedule it via cron for hands-off maintenance.

Why Automate Temp Cleanup?

  • Disk space — prevents runaway temp folders from filling your volume
  • Predictable storage — tiered retention aligns with backup and compliance windows
  • Zero overhead — find is fast, native, and requires no background daemon
  • Cron-ready — set it once in crontab and forget about it

The Solution

Use find with -mtime to target files older than a specific number of days:

# Delete files older than 30 days
find /home/scalan/tempo/30-days -type f -mtime +30 -delete

# Delete files older than 60 days
find /home/scalan/tempo/60-days -type f -mtime +60 -delete

# Delete files older than 90 days
find /home/scalan/tempo/90-days -type f -mtime +90 -delete

Each directory enforces its own retention policy. Files within the threshold are untouched; anything past the limit is removed instantly.

Understanding the Flags

FlagPurpose
/home/scalan/tempo/30-daysThe directory to scan (adjust to your temp path)
-type fMatch only regular files (skip directories and symlinks)
-mtime +30Select files modified more than 30 days ago
-deleteRemove matching files immediately

Note: -name "*" is unnecessary since find matches all files by default. It has been omitted for brevity.

How It Works

  1. Scan — find walks through every entry in the target directory
  2. Filter — only regular files (-type f) older than N days (-mtime +N) pass the test
  3. Delete — the -delete action removes each matching file on the spot

You can preview which files will be deleted before running with -delete:

# Dry run: list files that would be deleted
find /home/scalan/tempo/30-days -type f -mtime +30

Automate with Cron

Add these commands to a cron job for automatic daily cleanup:

# Run temp cleanup every day at 2:00 AM
0 2 * * * find /home/scalan/tempo/30-days -type f -mtime +30 -delete
0 2 * * * find /home/scalan/tempo/60-days -type f -mtime +60 -delete
0 2 * * * find /home/scalan/tempo/90-days -type f -mtime +90 -delete

Or wrap them in a script at /usr/local/bin/cleanup-temp.sh and trigger with a single cron entry:

0 2 * * * /usr/local/bin/cleanup-temp.sh

Customization Tips

  • Retention period — change +30 to any number of days that fits your policy (+7, +180, etc.)
  • Pattern matching — add -name "*.log" or -name "*.tmp" to target specific file types
  • Empty directories — append -depth -empty -delete to remove leftover empty folders after file deletion
  • Logging — redirect output for audit trails: find ... -print >> /var/log/temp-cleanup.log
  • Safety — test with ls -l /path/to/dir | wc -l before and after to verify counts

A simple, native way to keep your temp directories lean. Schedule it once, reclaim disk forever.
