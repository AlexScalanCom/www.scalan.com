Run Local ELK Stack with Docker Compose

06 Jul 2021 AlexS
bash linux docker DevOps

Setting up a local Elasticsearch cluster for development is essential for testing search APIs, debugging indexing pipelines, or prototyping Kibana dashboards — without touching production data. Docker Compose makes it possible to launch a multi-node Elasticsearch cluster with Kibana in a single command.

This guide shows you how to spin up a 3-node Elasticsearch cluster with Kibana locally and manage indices and documents via the REST API.

Why Run ELK Locally?

  • Safe experimentation — test index mappings, queries, and aggregations without risking production
  • Dashboard prototyping — build and iterate on Kibana visualizations before deploying to staging
  • API development — validate your application's Elasticsearch client integration end-to-end
  • Zero infrastructure cost — runs entirely on your laptop with bounded resource usage

Setup

Create a docker-compose.yml file that defines 3 Elasticsearch nodes and 1 Kibana instance:

version: "2.2"
services:
  es01:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.12.0
    container_name: es01
    environment:
      - node.name=es01
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es02,es03
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - xpack.security.enabled=false
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data01:/usr/share/elasticsearch/data
    ports:
      - 9200:9200
    networks:
      - elastic

  es02:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.12.0
    container_name: es02
    environment:
      - node.name=es02
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es01,es03
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - xpack.security.enabled=false
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data02:/usr/share/elasticsearch/data
    networks:
      - elastic

  es03:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.12.0
    container_name: es03
    environment:
      - node.name=es03
      - cluster.name=es-docker-cluster
      - discovery.seed_hosts=es01,es02
      - cluster.initial_master_nodes=es01,es02,es03
      - bootstrap.memory_lock=true
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - xpack.security.enabled=false
    ulimits:
      memlock:
        soft: -1
        hard: -1
    volumes:
      - data03:/usr/share/elasticsearch/data
    networks:
      - elastic

  kib01:
    image: docker.elastic.co/kibana/kibana:8.12.0
    container_name: kib01
    ports:
      - 5601:5601
    environment:
      ELASTICSEARCH_HOSTS: http://es01:9200
    networks:
      - elastic

volumes:
  data01:
  data02:
  data03:

networks:
  elastic:
    driver: bridge

Note: xpack.security.enabled=false disables authentication for local development. Never use this in production.

Start the Stack

Launch all containers in detached mode:

docker-compose up -d

Verify Kibana is running by opening http://localhost:5601 in your browser.

Stop and Clean Up

Shut down the stack and remove containers (volumes are preserved):

docker-compose down

To also delete the data volumes and start fresh:

docker-compose down -v

Essential Elasticsearch Commands

Check Cluster Health

curl http://localhost:9200/_cluster/health?pretty

A healthy 3-node cluster reports status: "green" with all shards assigned.

List Nodes

curl "http://localhost:9200/_cat/nodes?v"

Shows IP, heap usage, CPU load, and master node (*) for each cluster member.

List Indices

curl "http://localhost:9200/_cat/indices?v"

Displays health status, shard count, document count, and storage size per index.

Managing Documents with curl

Create an Index

curl -X PUT "http://localhost:9200/products?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "settings": {
      "number_of_shards": 2,
      "number_of_replicas": 1
    }
  }'

Insert a Document (Auto-ID)

curl -X POST "http://localhost:9200/products/_doc?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Product1",
    "price": 10
  }'

Insert or Replace Document (Custom ID)

curl -X PUT "http://localhost:9200/products/_doc/p200?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Product2",
    "price": 15
  }'

Get Document by ID

curl "http://localhost:9200/products/_doc/p200?pretty"

Update a Field (Partial Update)

curl -X POST "http://localhost:9200/products/_update/p200?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "doc": {
      "price": 19
    }
  }'

Delete a Document

curl -X DELETE "http://localhost:9200/products/_doc/p200?pretty"

Delete an Entire Index

curl -X DELETE "http://localhost:9200/products?pretty"

Tip: Append ?pretty to any request for formatted JSON output that's easy to read in the terminal.

How It Works

  1. Docker Compose starts 3 Elasticsearch containers on a shared elastic bridge network
  2. Node discovery — each node connects via discovery.seed_hosts and forms a cluster led by the elected master
  3. Data persistence — named volumes (data01, data02, data03) survive container restarts
  4. Kibana connects to es01:9200 and provides a web UI for exploring indices and building dashboards

Customization Tips

  • Adjust memory — change -Xms512m -Xmx512m based on your laptop's available RAM (1-2 GB per node is reasonable)
  • Add Logstash — include a logstash service in the compose file for data ingestion pipelines
  • Enable security — remove xpack.security.enabled=false and set up ELASTICSEARCH_PASSWORD for local auth testing
  • Reset cluster — run docker-compose down -v to wipe all data and start from scratch

A full search engine cluster on your laptop in one command. Spin it up, experiment freely, tear it down when done.
