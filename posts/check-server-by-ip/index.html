Check Server Reachability by IP and Port in Bash

02 Nov 2021 AlexS
bash linux DevOps

In CI/CD pipelines and automation scripts, blindly connecting to a remote server can cause unnecessary failures and wasted build time. Before launching an SSH session or deploying artifacts, it makes sense to verify that the target server is actually reachable on the expected port.

This guide shows how to perform a fast TCP connectivity check using nc (netcat) — a lightweight tool available on virtually every Linux distribution.

Why Pre-Check Connectivity?

  • Fail fast — detect unreachable servers before wasting time on lengthy deployments
  • Graceful error handling — log meaningful failures instead of cryptic SSH timeouts
  • Pipeline reliability — conditionally skip or retry steps based on server availability
  • Zero overhead — a single TCP handshake takes milliseconds

The Solution

Use nc -z to perform a quick port scan that exits immediately with a success or failure code:

server=192.168.1.1

if nc -z "$server" 22 2>/dev/null; then
    echo "✅ $server is reachable on port 22"
    # Proceed with SSH, deployment, etc.
else
    echo "❌ $server is NOT reachable on port 22"
    # Handle failure: alert, retry, or abort
fi

The -z flag tells netcat to scan for listening daemons without sending any data, making it the fastest way to test port connectivity.

Understanding the Flags

FlagPurpose
-zZero-I/O mode — only checks if the port is open, no data transfer
2>/dev/nullSuppresses error output for a clean script execution

You can also add a timeout to prevent hanging on unresponsive hosts:

# Timeout after 5 seconds
if nc -z -w 5 "$server" 22 2>/dev/null; then
    echo "✅ $server responded within 5 seconds"
else
    echo "❌ $server did not respond or timed out"
fi
Additional FlagPurpose
-w 5Sets a 5-second timeout for the connection attempt

Real-World Usage in Jenkins

Here's how this fits into a Jenkins pipeline script:

#!/bin/bash

SERVERS=("192.168.1.10" "192.168.1.11" "192.168.1.12")
PORT=22

for server in "${SERVERS[@]}"; do
    if nc -z -w 5 "$server" "$PORT" 2>/dev/null; then
        echo "🟢 $server is up — proceeding with deployment"
        ssh user@"$server" "your-command-here"
    else
        echo "🔴 $server is down — skipping"
        exit 1
    fi
done

Customization Tips

  • Check multiple ports — wrap the check in a loop to verify HTTP (80), HTTPS (443), or custom application ports
  • Retry logic — add a retry loop with exponential backoff for flaky network conditions
  • Alerting — integrate with Slack, email, or PagerDuty when a critical server is unreachable
  • Health endpoint — for HTTP services, replace nc with curl -sf http://$server/health | grep OK

A simple two-line check that saves hours of debugging failed deployments. Always verify before you connect.
