Encrypt and Base64-Encode a String with a Public Key in Java

05 Aug 2022 AlexS
java

Public key encryption enables you to safely transmit sensitive data when only the receiver holds the private decryption key. In Java, the javax.crypto.Cipher class combined with RSA provides a straightforward way to encrypt strings and encode the output as Base64 — making it safe for storage in logs, databases, or JSON payloads.

This guide shows how to build a compact encrypt-and-encode method using RSA/PKCS1 padding.

Why Public Key Encryption?

  • Asymmetric security — anyone with the public key can encrypt, but only the private key holder can decrypt
  • Safe transmission — share the public key openly without risking data exposure
  • Base64 output — binary ciphertext is encoded to ASCII-safe text for seamless storage and transfer
  • Zero secrets on sender side — no private key needed at the point of encryption

The Solution

Create a method that encrypts a plaintext string with an RSA public key and returns the Base64-encoded ciphertext:

public byte[] encryptAndEncode(String data)
        throws NoSuchAlgorithmException, NoSuchPaddingException,
        InvalidKeyException, IllegalBlockSizeException,
        BadPaddingException {
    Cipher cipher = Cipher.getInstance("RSA/ECB/PKCS1Padding");
    cipher.init(Cipher.ENCRYPT_MODE, publicKey);
    return Base64.getEncoder().encode(cipher.doFinal(data.getBytes()));
}

The publicKey field should be a valid java.security.PublicKey instance loaded from a PEM file or generated from your key pair.

Understanding the Cipher

ComponentPurpose
"RSA/ECB/PKCS1Padding"Specifies RSA algorithm with PKCS#1 v1.5 padding (standard for compatibility)
Cipher.ENCRYPT_MODEInitializes the cipher for encryption (as opposed to decryption)
publicKeyThe RSA public key object used to encrypt the data
cipher.doFinal(...)Processes the plaintext bytes and produces encrypted ciphertext
Base64.getEncoder().encode(...)Encodes binary ciphertext into a safe ASCII string

Note: data.getBytes() uses the platform default charset. For production code, prefer data.getBytes(StandardCharsets.UTF_8).

How It Works

  1. Initialize cipher — Cipher.getInstance("RSA/ECB/PKCS1Padding") creates an RSA encryption engine with PKCS#1 padding
  2. Set mode and key — cipher.init() binds the public key and configures encryption mode
  3. Encrypt — cipher.doFinal(data.getBytes()) transforms plaintext bytes into ciphertext bytes
  4. Encode — Base64.getEncoder().encode() converts binary ciphertext to a readable string representation

Loading the Public Key from PEM

To use this method in practice, load your public key from a .pem file:

public static PublicKey loadPublicKeyFromPem(String filePath)
        throws IOException, GeneralSecurityException {
    String pem = new String(Files.readAllBytes(Paths.get(filePath)));
    pem = pem.replace("-----BEGIN PUBLIC KEY-----", "")
             .replace("-----END PUBLIC KEY-----", "")
             .replaceAll("\\s+", "");

    byte[] keyBytes = Base64.getDecoder().decode(pem);
    KeyFactory keyFactory = KeyFactory.getInstance("RSA");
    X509EncodedKeySpec spec = new X509EncodedKeySpec(keyBytes);
    return keyFactory.generatePublic(spec);
}

Decrypting the Encrypted Data

The receiver uses the matching private key to reverse the process:

public String decryptAndDecode(byte[] encodedData)
        throws GeneralSecurityException {
    Cipher cipher = Cipher.getInstance("RSA/ECB/PKCS1Padding");
    cipher.init(Cipher.DECRYPT_MODE, privateKey);
    byte[] decoded = Base64.getDecoder().decode(encodedData);
    byte[] plaintext = cipher.doFinal(decoded);
    return new String(plaintext, StandardCharsets.UTF_8);
}

Customization Tips

  • OAEP padding — for stronger security, replace "RSA/ECB/PKCS1Padding" with "RSA/ECB/OAEPWithSHA-256AndMGF1Padding"
  • Large payloads — RSA has a maximum input size (key length minus padding overhead); for larger data, encrypt an AES session key instead of the payload directly
  • Charset safety — always specify StandardCharsets.UTF_8 to avoid platform-dependent encoding issues
  • Exception handling — wrap checked exceptions in a custom unchecked exception to simplify method signatures

A single method that locks your data with asymmetric encryption. Encrypt on the sender side, decrypt only where the private key lives.
