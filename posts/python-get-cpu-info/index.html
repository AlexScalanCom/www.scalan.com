Python Get CPU Temperature and Render It on an Image

27 Feb 2021 AlexS
python linux raspberryPi

On a Raspberry Pi running a custom dashboard or clock display, showing the CPU temperature provides at-a-glance hardware health monitoring. The psutil library reads thermal data from /sys/class/thermal/, and Pillow (PIL) renders the value as rotated text directly onto your image canvas — no HTML overlay needed.

This guide shows how to fetch the CPU temperature in Python and draw it vertically on an existing image.

Why Display CPU Temperature on Screen?

  • Hardware monitoring — spot overheating or thermal throttling without SSH access
  • Kiosk mode support — Raspberry Pi dashboards often run full-screen with no terminal visible
  • Zero extra hardware — reads from the built-in SoC thermal sensor via psutil
  • Image-native rendering — Pillow bakes text into the canvas, avoiding compositing issues

The Solution

from psutil.sensors import CPUTemperature
from PIL import Image, ImageDraw, ImageFont

# Get CPU temperature
cpu = CPUTemperature()
degree_sign = u"\N{DEGREE SIGN}"
text = str(int(cpu.temperature)) + degree_sign + "C"

# Create font and render text to a transparent image
font = ImageFont.truetype('./freefont/FreeMonoBold.ttf', 22)
img_txt = Image.new('L', font.getsize(text), 255)
draw_txt = ImageDraw.Draw(img_txt)
draw_txt.text((0, 0), text, font=font, fill=0)

# Rotate 90° for vertical display and paste onto main canvas
image_width = font.getsize(text)
t = img_txt.rotate(90, expand=1)
image.paste(t, (155, 260 - image_width[0]))

Required Dependencies

Install the required packages:

pip3 install psutil Pillow

Make sure you have a TrueType font file (e.g., FreeMonoBold.ttf) in your project directory.

Understanding the Code

ComponentPurpose
CPUTemperature()Reads CPU thermal sensor data from /sys/class/thermal/
cpu.temperatureReturns current CPU temperature in degrees Celsius as a float
u"\N{DEGREE SIGN}"Unicode degree symbol (°) for proper temperature formatting
ImageFont.truetype()Loads a TrueType font at the specified size (22pt)
font.getsize(text)Returns (width, height) tuple needed for image dimensions and positioning
Image.new('L', ...)Creates an 8-bit grayscale image sized to fit the text string
ImageDraw.Draw()Provides drawing methods (text, line, rectangle) on the image
.rotate(90, expand=1)Rotates image 90° clockwise; expand=1 resizes canvas to fit rotated content
image.paste()Pastes the rotated temperature label onto the main canvas at (x, y)

How It Works

  1. Read sensor — CPUTemperature() queries the kernel thermal zone and returns degrees Celsius
  2. Format string — rounds to integer, appends °C symbol (e.g., "58°C")
  3. Render text — creates a small grayscale image sized exactly to the text, draws white background with black lettering
  4. Rotate 90° — turns the horizontal label into vertical text for sidebar display
  5. Paste onto canvas — places the rotated temperature label at pixel coordinates (155, 260 - height) on the main image

Practical Examples

Show Temperature Horizontally

Remove rotation for a standard horizontal label:

draw_txt.text((10, 250), text, font=font, fill=(255, 255, 255))

Color-Code by Threshold

Change text color based on temperature levels:

temp = int(cpu.temperature)
if temp < 50:
    color = (0, 255, 0)   # green - safe
elif temp < 70:
    color = (255, 255, 0) # yellow - warm
else:
    color = (255, 0, 0)   # red - hot

Update in a Loop for Live Display

import time

while True:
    cpu_temp = int(CPUTemperature().temperature)
    text = f"{cpu_temp}\N{DEGREE SIGN}C"
    # ... render and show image
    time.sleep(5)  # update every 5 seconds

Customization Tips

  • Modern font sizing — replace font.getsize() with font.getbbox(text) for Pillow ≥10 compatibility (getsize is deprecated)
  • Font fallback — use ImageFont.load_default() if no .ttf file is available
  • Multi-core temps — iterate over cpu.temp list (if available) to show per-core temperatures
  • Fan control trigger — combine with GPIO pins to activate a cooling fan when temp exceeds 75°C

One sensor read, one image render. The temperature baked into your dashboard, always visible, never throttled.
