Wait for AWS Elastic Beanstalk Environment to Be Ready

19 Dec 2020 AlexS
bash linux DevOps

Deploying to AWS Elastic Beanstalk is not instantaneous. After triggering a deployment via eb deploy or through CI/CD pipelines, the environment enters an Launching state that can take several minutes to resolve. Running subsequent steps — health checks, database migrations, or cache warming — before the environment is ready will fail silently or throw connection errors. A simple bash polling loop with the AWS CLI solves this.

This guide shows how to wait for an Elastic Beanstalk environment to reach Ready status before continuing your deployment script.

Why Wait for Beanstalk Ready?

  • Prevent race conditions — avoid running post-deploy tasks while the platform is still provisioning
  • CI/CD pipeline reliability — ensure each step completes before the next one starts
  • Fail-fast behavior — detect deployments that never reach Ready state within a reasonable timeout
  • Zero extra dependencies — uses only bash and the standard AWS CLI, no external tools required

The Solution

#!/bin/bash
echo 'Waiting...'
status=''
iterations=0
maxiterations=20

while [[ ( $status != "Ready" ) && ( $iterations -lt $maxiterations ) ]]; do
  status=$(aws elasticbeanstalk describe-environments \
    --region us-east-1 \
    --environment-name app-env \
    --output text \
    --query 'Environments[0].Status')
  echo $status
  sleep 15
  ((iterations+=1))
done

echo "Done."

Replace us-east-1 with your target region and app-env with your actual environment name. The loop polls every 15 seconds for up to 20 iterations (5 minutes total), exiting early once status becomes Ready.

Understanding the Command

ComponentPurpose
aws elasticbeanstalk describe-environmentsAWS CLI command that queries Elastic Beanstalk environment metadata
--region us-east-1Specifies the AWS region where the environment is deployed
--environment-name app-envFilters results to a specific environment by name
--output textReturns plain text output instead of JSON for easy parsing in bash
--query 'Environments[0].Status'JMESPath query that extracts only the Status field from the first environment
while [[ ... ]]Bash conditional loop that continues while status is not "Ready" and iterations haven't exceeded the maximum
sleep 15Waits 15 seconds between polls to avoid throttling AWS API rate limits
((iterations+=1))Increments counter on each loop iteration to enforce timeout

How It Works

  1. Initialize variables — status starts empty, iterations starts at 0, maxiterations caps at 20 (5-minute timeout)
  2. Evaluate condition — loop continues while status is not "Ready" AND iterations are below the maximum
  3. Poll AWS API — describe-environments fetches current status via the AWS CLI with JMESPath filtering
  4. Print progress — echoes the current status so you can track it in CI/CD logs or terminal output
  5. Wait and repeat — sleeps 15 seconds, increments counter, re-evaluates condition
  6. Exit loop — stops when status becomes "Ready" or max iterations reached

Practical Examples

Make Environment Name Configurable

Pass it as a script argument:

#!/bin/bash
ENV_NAME="${1:-app-env}"
REGION="${2:-us-east-1}"

status=''
iterations=0
maxiterations=20

while [[ ( $status != "Ready" ) && ( $iterations -lt $maxiterations ) ]]; do
  status=$(aws elasticbeanstalk describe-environments \
    --region "$REGION" --environment-name "$ENV_NAME" \
    --output text --query 'Environments[0].Status')
  echo "$status"
  sleep 15
  ((iterations+=1))
done

Usage: ./wait-beanstalk.sh my-production-env eu-west-1

Fail Script If Timeout Reached

Exit with error code if environment didn't become ready in time:

if [[ $status != "Ready" ]]; then
  echo "ERROR: Environment did not become Ready within $((maxiterations * 15)) seconds"
  exit 1
fi
echo "Environment is Ready, continuing deployment..."

Wait for Specific Health Status

Check Health field instead of Status (requires environment to be running first):

health=$(aws elasticbeanstalk describe-environments \
  --region us-east-1 --environment-name app-env \
  --output text --query 'Environments[0].Health')

while [[ $health != "Green" ]]; do
  health=$(aws elasticbeanstalk describe-environments \
    --region us-east-1 --environment-name app-env \
    --output text --query 'Environments[0].Health')
  echo "Health: $health"
  sleep 30
done

Use in GitHub Actions Workflow

- name: Wait for Beanstalk Ready
  run: |
    status=''
    for i in $(seq 1 20); do
      status=$(aws elasticbeanstalk describe-environments \
        --region us-east-1 --environment-name app-env \
        --output text --query 'Environments[0].Status')
      [[ "$status" == "Ready" ]] && break
      sleep 15
    done
    [[ "$status" != "Ready" ]] && echo "FAILED" && exit 1

Customization Tips

  • Adjust polling interval — increase sleep to 30 seconds if you hit AWS API throttling, or decrease to 5 seconds for faster feedback on quick deployments
  • Increase timeout — set maxiterations=60 with sleep 15 for a 15-minute wait window on large application deployments
  • Add event notifications — pipe status changes to Slack or email using curl to a webhook URL when status transitions from Launching to Ready
  • Profile-based auth — add --profile my-aws-profile to the AWS CLI command if you use named IAM profiles

One polling loop, one API query repeated. Beanstalk reaches Ready or the timeout saves your pipeline from hanging forever.
