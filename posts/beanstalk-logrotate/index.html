AWS Elastic Beanstalk — Configure Nginx Log Rotation

18 Mar 2021 AlexS
bash linux aws DevOps

By default, Nginx logs on AWS Elastic Beanstalk grow indefinitely. Without log rotation, your disk fills up quickly and your application crashes. The good news is that Elastic Beanstalk makes it trivial to deploy custom system configuration through .ebextensions.

This guide shows you how to automatically rotate Nginx logs on every deployment — no manual SSH required.

Why Log Rotation?

  • Disk space — prevents log files from consuming all available storage
  • Easy debugging — smaller, date-stamped files are easier to search and manage
  • Compliance — many teams require bounded log retention periods
  • Zero maintenance — logrotate runs via cron automatically after deployment

Setup

Create a new file at .ebextensions/logrotate.config in your application source bundle:

files:
  "/etc/logrotate.d/nginx":
    mode: "000644"
    content: |
      /var/log/nginx/*.log {
        daily
        compress
        delaycompress
        rotate 30
        dateformat -%Y-%m-%d
        missingok
        sharedscripts
        postrotate
          test ! -f /var/run/nginx.pid || kill -USR1 `cat /var/run/nginx.pid`
        endscript
      }

On the next deployment, Elastic Beanstalk will automatically place this configuration into /etc/logrotate.d/nginx, and the system's cron job will start rotating logs daily.

Configuration Breakdown

DirectiveWhat It Does
/var/log/nginx/*.logTargets all Nginx log files in the default directory
dailyRotate logs every day
compressCompress rotated logs with gzip to save disk space
delaycompressDelay compression by one cycle (keeps most recent rotated log readable)
rotate 30Keep 30 days of rotated logs before deleting oldest
dateformat -%Y-%m-%dAppend date stamp to rotated filenames (e.g., access.log-2024-09-07)
missingokDon't throw errors if a log file is missing
sharedscriptsRun the postrotate script only once across all matched files
postrotate ... endscriptSend USR1 signal to Nginx to reopen log file handles after rotation

How It Works

  1. Deployment — when you push a new version to Elastic Beanstalk, the .ebextensions hook writes the logrotate config to the instance
  2. Daily cron — the system's built-in /etc/cron.daily/logrotate runs automatically and applies the rotation rules
  3. Nginx signal — after rotation, Nginx receives a USR1 signal and starts writing to fresh log files

You can manually trigger rotation at any time:

sudo logrotate -f /etc/logrotate.d/nginx

Verify rotated files:

ls -lh /var/log/nginx/

Customization Tips

  • Retention period — adjust rotate 30 to keep more or fewer days of history
  • Rotation frequency — change daily to weekly or monthly for less frequent rotation
  • Size-based rotation — add size 100M to rotate when files exceed a certain size regardless of time
  • Custom log paths — update the glob pattern if your Nginx logs reside elsewhere

A one-file fix that keeps your Elastic Beanstalk instances clean and prevents disk-space emergencies. Deploy it once and forget about it.
