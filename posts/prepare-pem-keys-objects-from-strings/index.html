Java Parse PEM Keys from Strings with Bouncy Castle

01 Aug 2022 AlexS
java

License files and credential bundles often pack multiple PEM blocks — certificates, RSA private keys, and JWT tokens — into a single string. To verify signatures or decrypt payloads in Java, you need to extract each block, convert it into usable PublicKey and PrivateKey objects, and optionally decode the embedded JWT claims. Bouncy Castle makes this straightforward with its PEMParser and JcaPEMKeyConverter.

This guide shows how to parse a PEM string, extract keys, and read JWT data stored inside a license file.

Why Parse PEM Keys from Strings?

  • License verification — validate signed JWT tokens embedded in enterprise license files
  • Runtime key loading — avoid filesystem dependencies by embedding credentials as string resources
  • Flexible key formats — handle both RSA PRIVATE KEY and generic PRIVATE KEY wrappers transparently
  • JWT claim extraction — decode payload data without external tools or manual base64 decoding

The Solution

Extract PEM Blocks with a Helper Method

String licensePem = "..."; // Read pem file into this string variable.

String certificateStr = between(licensePem, "-----BEGIN CERTIFICATE-----", "-----END CERTIFICATE-----");
String privateKeyStr = between(licensePem, "-----BEGIN RSA PRIVATE KEY-----", "-----END RSA PRIVATE KEY-----");

// Fallback to generic private key format if RSA not found
if (privateKeyStr == null) {
    privateKeyStr = between(licensePem, "-----BEGIN PRIVATE KEY-----", "-----END PRIVATE KEY-----");
}

// Extract JWT license token (try both Service and Enterprise formats)
String licenseJwtStr = between(licensePem, "-----BEGIN SERVICE LICENSE-----", "-----END SERVICE LICENSE-----");
if (licenseJwtStr == null) {
    licenseJwtStr = betweenSharp(licensePem, "-----BEGIN ENTERPRISE LICENSE-----", "-----END ENTERPRISE LICENSE-----");
}
if (licenseJwtStr != null) {
    licenseJwtStr = licenseJwtStr.replaceAll("\n", StringUtils.EMPTY);
}

Parse Public Key from Certificate String

Security.addProvider(new BouncyCastleProvider());

PublicKey publicKey = null;
try (PEMParser parser = new PEMParser(new StringReader(certificateStr))) {
    Object keyInfo = parser.readObject();
    if (keyInfo instanceof X509CertificateHolder) {
        publicKey = new JcaPEMKeyConverter().setProvider("BC")
            .getPublicKey(((X509CertificateHolder) keyInfo).getSubjectPublicKeyInfo());
    } else {
        publicKey = new JcaPEMKeyConverter().setProvider("BC")
            .getPublicKey((SubjectPublicKeyInfo) keyInfo);
    }
} catch (IOException ex) {
    log.error("Failed to parse public key: {}", ex.getMessage());
}

Parse Private Key from String

PrivateKey privateKey = null;
try (PEMParser parser = new PEMParser(new StringReader(privateKeyStr))) {
    Object keyInfo = parser.readObject();
    if (keyInfo instanceof PEMKeyPair) {
        privateKey = new JcaPEMKeyConverter().setProvider("BC")
            .getPrivateKey(((PEMKeyPair) keyInfo).getPrivateKeyInfo());
    }
} catch (IOException ex) {
    log.error("Failed to parse private key: {}", ex.getMessage());
}

Decode JWT Claims

Map<String, Object> dataMap = null;
try {
    JwtConsumer jwtConsumer = new JwtConsumerBuilder()
        .setRequireExpirationTime()
        .setVerificationKey(publicKey)
        .build();

    JwtClaims jwtDecoded = jwtConsumer.processToClaims(licenseJwtStr);
    dataMap = jwtDecoded.getClaimsMap();
} catch (InvalidJwtException ex) {
    log.error("JWT validation failed: {}", ex.getMessage());
}

// Use data stored in the dataMap object

String Extraction Utilities

Two helper methods for extracting content between PEM markers:

// Returns content including delimiters (BEGIN...END tags included)
public static String between(String source, String start, String end) {
    int startPos = source.indexOf(start);
    if (startPos < 0) return null;
    int endPos = source.indexOf(end);
    if (endPos < 0) return null;
    return source.substring(startPos, endPos + end.length());
}

// Returns content excluding delimiters (inner content only)
public static String betweenSharp(String source, String start, String end) {
    int startPos = source.indexOf(start);
    if (startPos < 0) return null;
    int endPos = source.indexOf(end);
    if (endPos < 0) return null;
    return source.substring(startPos + start.length(), endPos);
}

Required Dependencies

Add Bouncy Castle and JWT library to your pom.xml:

<dependency>
    <groupId>org.bouncycastle</groupId>
    <artifactId>bcpkix-jdk18on</artifactId>
    <version>1.78.1</version>
</dependency>
<dependency>
    <groupId>com.nimbusds</groupId>
    <artifactId>nimbus-jose-jwt</artifactId>
    <version>9.37.3</version>
</dependency>

Understanding the Components

ComponentPurpose
Security.addProvider(new BouncyCastleProvider())Registers BC as a crypto provider for key conversion
PEMParserReads PEM-encoded objects from a character stream
JcaPEMKeyConverterConverts BC key info objects to standard java.security.Key instances
X509CertificateHolderHolds X.509 certificate data extracted from the PEM block
PEMKeyPairHolds private/public key pair parsed from PEM format
JwtConsumerNimbus library class that validates and decodes JWT tokens
between()Extracts substring including PEM delimiters (for certificate parsing)
betweenSharp()Extracts substring excluding PEM delimiters (for raw JWT payload)

How It Works

  1. Block extraction — helper methods locate and extract each PEM section (certificate, private key, JWT license) from the combined string
  2. Format fallback — checks for both RSA PRIVATE KEY and generic PRIVATE KEY wrappers, plus both Service and Enterprise license formats
  3. PEM parsing — PEMParser reads the certificate and key strings, producing intermediate objects (X509CertificateHolder, PEMKeyPair)
  4. Key conversion — JcaPEMKeyConverter with BC provider transforms intermediate objects into standard Java PublicKey and PrivateKey
  5. JWT decoding — JwtConsumer verifies the JWT signature using the extracted public key and returns the claims map

Customization Tips

  • KeyStore storage — wrap extracted keys into a KeyStore for reuse across multiple service calls
  • Refresh on expiration — check exp claim from the JWT and reload the license file before token expiry
  • ASN.1 direct parsing — for non-PEM encoded keys, use SubjectPublicKeyInfo.getInstance() with raw DER bytes
  • PKCS#8 support — the same PEMParser approach works for PKCS#8 formatted private keys without code changes
