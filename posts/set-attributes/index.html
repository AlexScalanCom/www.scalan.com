Set All Directories to 755 and Files to 644 with find

27 Sep 2024 AlexS
bash linux DevOps

Deploying a web application or preparing source code for distribution often requires consistent file permissions. Directories need the execute bit so they can be traversed, while files should remain readable without being world-writable. Two find commands with chmod handle this recursively across an entire tree in seconds.

This guide shows how to set all directories to 755 and all files to 644 from a single starting point.

Why Standardize Permissions?

  • Security baseline — prevent world-writable files (777) that expose data or allow code injection
  • Web server compatibility — Apache and Nginx require readable files (644) and traversable directories (755)
  • Deployment hygiene — ensure consistent permissions after copying, extracting archives, or cloning repos
  • Automated enforcement — run before deployment to guarantee the expected permission structure

The Solution

Run two find commands from your target directory:

# Set all directories to 755 (rwxr-xr-x)
find . -type d -exec chmod 0755 {} \;

# Set all files to 644 (rw-r--r--)
find . -type f -exec chmod 0644 {} \;

Execute these from the root of your project or deployment directory. The . means "start searching here."

Understanding the Command

ComponentPurpose
findRecursively traverses the directory tree starting from the given path
.Current directory — the search root (replace with any path)
-type dMatch only directories (use -type f for regular files)
-exec chmod 0755 {} \;Execute chmod 0755 on each matched item; {} is replaced by the found path, \; terminates the command
0755Directory permission: owner full control (rwx), group read+execute (r-x), others read+execute (r-x)
0644File permission: owner read+write (rw-), group read-only (r--), others read-only (r--)

How It Works

  1. Traverse tree — find . walks the entire directory structure starting from the current path
  2. Filter by type — -type d selects only directories; -type f selects only regular files (not symlinks or sockets)
  3. Execute chmod — for each match, -exec runs chmod with the specified octal permission on that item
  4. Apply permissions — directories get 755 (traversable), files get 644 (readable but not executable)

Practical Examples

Target a Specific Directory

find /var/www/myapp -type d -exec chmod 0755 {} \;
find /var/www/myapp -type f -exec chmod 0644 {} \;

Make Specific Files Executable After Setting Base Permissions

Set baseline, then add execute bit to scripts:

find . -type f -exec chmod 0644 {} \;
find . -name "*.sh" -type f -exec chmod 0755 {} \;
find . -name "Makefile" -type f -exec chmod 0755 {} \;

Dry Run — Preview What Will Change

Before applying, see which items would be affected:

find . -type d -ls   # List all directories that will get 755
find . -type f -ls   # List all files that will get 644

Exclude Node Modules or Vendor Directories

find . -path ./node_modules -prune -o -type f -exec chmod 0644 {} \;
find . -path ./vendor -prune -o -type d -exec chmod 0755 {} \;

Customization Tips

  • Use + instead of \; — replace \; with + to batch multiple paths into a single chmod call, significantly speeding up large trees: find . -type f -exec chmod 0644 {} +
  • Set umask before extraction — run umask 0022 before untarring archives so files land with correct permissions automatically
  • Include symlinks — use -type l if you need to handle symbolic links (note: chmod on symlinks typically has no effect on the link itself)
  • Verify with stat — after running, check a sample: stat -c "%a %n" ./* shows octal permissions for each entry

Two find commands that lock down your entire tree. Directories traversable, files readable, nothing writable by strangers.
