Fix Linux Konsole Color Scheme with Material Theme

31 Aug 2023 AlexS
bash linux DevOps

The default Konsole terminal theme on Linux can be harsh — washed-out colors, poor contrast, or jarring bright backgrounds. Applying a custom Material Design color scheme transforms your terminal into a clean, readable workspace with balanced foreground and background tones.

This guide shows how to create a Material.colorscheme file and apply it to your Konsole profile.

Why Customize Your Terminal Theme?

  • Reduced eye strain — darker backgrounds with muted tones are gentler during long sessions
  • Better contrast — carefully chosen color pairs make text pop without blowing out highlights
  • Consistent palette — Material Design colors provide a familiar, well-tested spectrum
  • Zero dependency — a simple config file in your home directory, no packages to install

The Solution

Create the color scheme file at ~/.local/share/konsole/Material.colorscheme with the following content:

[Background]
Color=0,0,0

[BackgroundIntense]
Color=55,71,79

[Color0]
Color=0,0,0

[Color0Intense]
Color=55,71,79

[Color1]
Color=183,28,28

[Color1Intense]
Color=244,67,54

[Color2]
Color=100,221,23

[Color2Intense]
Color=118,255,3

[Color3]
Color=245,124,0

[Color3Intense]
Color=255,235,59

[Color4]
Color=19,28,97

[Color4Intense]
Color=33,150,243

[Color5]
Color=170,0,255

[Color5Intense]
Color=224,64,251

[Color6]
Color=0,188,212

[Color6Intense]
Color=0,229,255

[Color7]
Color=178,178,178

[Color7Intense]
Color=255,255,255

[Foreground]
Color=176,190,197

[ForegroundIntense]
Color=255,255,255

[General]
Description=Material
Opacity=1
Wallpaper=

Then open Konsole → Settings → Edit Current Profile → Appearance, and select Material from the color scheme dropdown.

Understanding the Color Palette

SectionPurposeMaterial Mapping
Background / ForegroundPrimary canvas and text colorsPure black bg, soft gray fg
BackgroundIntense / ForegroundIntenseBold and reversed backgroundsDark gray / pure white
Color0–Color7ANSI base colors (black → white)Material dark shades
ColorXIntenseANSI bright variantsMaterial vibrant tones
Color1 (Red)Errors, deletions, alertsMaterial Red 900/500
Color2 (Green)Success states, file permissionsMaterial Green 900/400
Color3 (Orange/Yellow)Warnings, cautionsMaterial Orange 800/Yellow 400
Color4 (Blue)Links, info messagesMaterial Blue 950/400
Color5 (Purple)Special markers, syntax highlightsMaterial Purple 900/300
Color6 (Cyan)Timestamps, secondary infoMaterial Cyan 800/200

How It Works

  1. File creation — Konsole scans ~/.local/share/konsole/ for .colorscheme files on startup
  2. INI parsing — each [Section] with a Color=R,G,B value maps to an ANSI or UI color slot
  3. Profile binding — selecting the scheme in profile settings applies it to all new terminal windows
  4. Runtime effect — existing sessions pick up the theme on next launch; no reboot required

Practical Examples

Apply via Command Line (Quick Setup)

mkdir -p ~/.local/share/konsole/

# Paste the colorscheme content
cat > ~/.local/share/konsole/Material.colorscheme << 'EOF'
[Background]
Color=0,0,0
...
EOF

Verify Installation

List available color schemes to confirm Konsee detected the file:

ls ~/.local/share/konsole/*.colorscheme

Copy to Multiple Machines

Drop the Material.colorscheme file into your dotfiles repository for instant deployment on any new Linux workstation.

Customization Tips

  • Adjust opacity — change Opacity=1 to Opacity=0.9 for slight transparency with a wallpaper behind
  • Tweak individual colors — edit any Color=R,G,B value and reload Konsole to see changes instantly
  • Add a wallpaper — set Wallpaper=/path/to/background.png in the [General] section
  • Share across profiles — the scheme file is global; apply it to multiple Konsole profiles independently

A single config file that transforms your terminal from default drab to Material sharp. Drop it in, select it, done.
