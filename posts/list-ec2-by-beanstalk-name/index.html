List EC2 Instance IDs by Elastic Beanstalk Environment Name

19 Oct 2021 AlexS
bash linux DevOps

AWS Elastic Beanstalk abstracts your infrastructure, but sometimes you need the actual EC2 instance IDs — for SSH access, log inspection, or attaching monitoring tools. The AWS CLI provides a one-liner that queries any Beanstalk environment and returns the underlying instance IDs without manually browsing the console.

This guide shows how to extract EC2 instance IDs from an Elastic Beanstalk environment by name.

Why Query Instance IDs via CLI?

  • Automation — feed instance IDs into scripts for batch operations (stop, restart, tag)
  • SSH access — quickly grab instance IDs to connect via ssh -i key.pem ec2-user@<ip>
  • Debugging — identify which EC2 machines back a specific Beanstalk environment
  • CI/CD integration — retrieve instances programmatically in deployment pipelines

The Solution

Use aws elasticbeanstalk describe-environment-resources with a JMESPath query to extract instance IDs:

envname="wp-env"
for ec2Id in $(aws elasticbeanstalk describe-environment-resources \
  --environment-name "$envname" \
  --output text \
  --query 'EnvironmentResources.Instances[*].Id'); do
  echo "$ec2Id"
done

Replace "wp-env" with your actual Elastic Beanstalk environment name. Each instance ID is printed on a separate line.

Understanding the Command

ComponentPurpose
aws elasticbeanstalkAWS CLI service namespace for Elastic Beanstalk operations
describe-environment-resourcesAPI call that returns all resources (instances, load balancers, Auto Scaling groups) for an environment
--environment-name "$envname"Targets a specific Beanstalk environment by its name
--output textReturns results as tab-separated text instead of JSON
--query 'EnvironmentResources.Instances[*].Id'JMESPath expression that extracts only the EC2 instance IDs from the response
for ec2Id in $(...)Shell loop that iterates over each returned instance ID

Prerequisite: AWS CLI must be installed and configured with valid credentials (aws configure). Your account needs the elasticbeanstalk:DescribeEnvironmentResources permission.

How It Works

  1. API call — describe-environment-resources queries AWS for all resources backing the named environment
  2. JMESPath filter — the --query expression navigates the JSON tree to EnvironmentResources.Instances[*].Id, extracting only instance IDs
  3. Text output — --output text flattens the result into space-separated values
  4. Shell iteration — the for loop processes each ID individually, allowing further commands per instance
