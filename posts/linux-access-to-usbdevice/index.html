Fix Linux /dev/ttyUSB0 Permission Denied Error

21 Sep 2021 AlexS
bash linux arduino

Connecting an Arduino board or any USB serial device to a Linux machine often results in a frustrating Permission denied error when trying to access /dev/ttyUSB0. This happens because your user account lacks the necessary group permissions to communicate with serial and plug-in hardware devices.

This guide shows how to grant access by adding your user to the dialout and plugdev groups.

Why Does Permission Denied Occur?

  • Restricted device nodes — /dev/ttyUSB0 is owned by root with group dialout, blocking regular users
  • No plug-in device access — USB devices are managed by udev rules that restrict access to the plugdev group
  • Fresh installations — new Linux setups don't automatically add users to hardware-access groups
  • Multiple group requirement — some distributions require both dialout and plugdev for full serial device access

The Solution

Add your user account to the dialout and plugdev groups by running these commands with sudo:

sudo usermod -a -G dialout $USER
sudo usermod -a -G plugdev $USER

Replace $USER with your actual username if needed (e.g., scalan). After executing, log out and log back in (or reboot) for the group changes to take effect.

Understanding the Flags

FlagPurpose
usermodModifies an existing user account's properties
-aAppend — adds the user to supplementary groups without removing existing ones
-G dialoutSpecifies the dialout group, which grants access to serial ports (/dev/ttyUSB*)
-G plugdevSpecifies the plugdev group, which grants access to removable and plug-in hardware
$USEREnvironment variable that expands to your current username

Important: The -a flag is critical. Without it, usermod -G replaces all supplementary groups instead of appending, potentially breaking your existing permissions.

How It Works

  1. usermod execution — adds your user to the specified groups in /etc/group
  2. Group reload — logging out and back in applies the new group memberships to your session
  3. udev recognition — the kernel's device manager grants your user access to matching device nodes based on group membership
  4. Device access — your applications can now open /dev/ttyUSB0 without sudo

Practical Examples

Verify Group Membership

Confirm your user was added successfully:

groups $USER
# Output should include: dialout plugdev

Test Device Access

Check permissions on the USB device node:

ls -l /dev/ttyUSB0
# crw-rw---- 1 root dialout 188, 0 ...

Try reading from the device:

cat /dev/ttyUSB0

Arduino-Specific Test

Open the Arduino IDE and verify your board appears in Tools → Port without permission errors.

Customization Tips

  • Apply to another user — replace $USER with the target username: sudo usermod -a -G dialout otheruser
  • Skip reboot — run newgrp dialout in your current terminal session to activate the group without a full logout
  • Custom udev rules — for non-standard devices, create /etc/udev/rules.d/99-usb.rules with custom GROUP assignments
  • Lockdown access — remove users from groups with gpasswd -d username dialout when revoking device access
