Setup FTP Server on Raspberry Pi with vsftpd

27 Sep 2024 AlexS
bash linux DevOps raspberryPi

Raspberry Pi makes a lightweight file server for small networks or IoT deployments. vsftpd (Very Secure FTP Daemon) is the go-to FTP server on Debian-based systems — it's fast, well-tested, and secure by default. Configuring local user access with a dedicated FTP directory takes just a few minutes.

This guide walks through installing vsftpd, securing the configuration for single-user access, and creating a chroot jail so the FTP user can only reach their designated folder.

Why Use vsftpd on Raspberry Pi?

  • Lightweight footprint — minimal CPU and memory usage, ideal for ARM-based SBCs
  • Secure by design — chroot jails, TLS support, and per-user configuration out of the box
  • Simple setup — one config file controls all behavior, no database or daemon complexity
  • Reliable transfer protocol — FTP works with legacy tools, embeddable devices, and simple upload scripts

The Solution

Install vsftpd

sudo apt update
sudo apt install vsftpd -y

Create FTP Directory Structure

Replace [user] with your actual Linux username:

mkdir -p /home/[user]/ftp
chmod a-w /home/[user]/ftp

The a-w (remove write for all) on the root FTP directory is required by vsftpd's security model — the chroot directory must not be writable by the user, but a subdirectory inside it can be.

Create Writable Upload Directory

mkdir -p /home/[user]/ftp/files
chown [user]:[user] /home/[user]/ftp/files

Configure vsftpd

Edit the main configuration file:

sudo nano /etc/vsftpd.conf

Uncomment or add these settings (remove leading # if present):

write_enable=YES
local_umask=022
anonymous_enable=NO
user_sub_token=$USER
local_root=/home/$USER/ftp

Restart the Service

sudo service vsftpd restart

Verify it's running:

sudo service vsftpd status

Understanding the Configuration

SettingPurpose
write_enable=YESAllow authenticated users to upload and modify files
local_umask=022Set file creation mask: files get 644, directories get 755
anonymous_enable=NODisable anonymous access — only authenticated local users can connect
user_sub_token=$USERReplace $USER in other directives with the logged-in username
local_root=/home/$USER/ftpChroot each user to their own FTP directory (e.g., /home/alice/ftp)

How It Works

  1. Install package — apt install vsftpd pulls the daemon and default config from Debian repositories
  2. Create FTP root — /home/[user]/ftp becomes the chroot jail for that user
  3. Remove write on root — chmod a-w satisfies vsftpd's security check (chroot dir must not be writable)
  4. Add writable subdirectory — /home/[user]/ftp/files is where uploads actually go, owned by the user
  5. Configure access — edit /etc/vsftpd.conf to enable writes, disable anonymous, and set the chroot path
  6. Restart daemon — service vsftpd restart applies new configuration

Practical Examples

Connect from Another Machine

ftp 192.168.1.100
# Enter username and password when prompted

Test Upload with curl

curl -T testfile.txt ftp://192.168.1.100/files/ --user [user]:[password]

Add a Second FTP User

sudo adduser newuser
mkdir -p /home/newuser/ftp/files
chmod a-w /home/newuser/ftp
chown newuser:newuser /home/newuser/ftp/files

The same vsftpd.conf with $USER token automatically applies to any local user.

Enable Passive Mode for Firewalled Networks

Add to /etc/vsftpd.conf:

pasv_min_port=30000
pasv_max_port=31000

Then open those ports in your firewall:

sudo ufw allow 30000:31000/tcp

Customization Tips

  • Enable FTPS (TLS) — generate a self-signed cert with openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/vsftpd.pem -out /etc/ssl/private/vsftpd.pem and add ssl_enable=YES to config
  • Restrict users to FTP only — set chroot_local_user=YES so they cannot SSH into the system, only use FTP
  • Log transfers — enable xferlog_enable=YES and check /var/log/vsftpd.log for upload/download records
  • Limit bandwidth — add local_max_rate=50000 (bytes/sec) to throttle uploads on slow connections

One package install, one config edit. FTP up, secured, and serving files from your Pi.
