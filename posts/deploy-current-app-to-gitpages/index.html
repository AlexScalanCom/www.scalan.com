Keeping a GitHub Repository Small by Resetting History

When publishing generated websites or static projects, Git history can become unnecessarily large because every deployment creates another commit containing almost the entire build output.

This deployment script replaces the repository history with a single commit on every deployment.

How it works

The script performs the following steps:

  1. Creates an orphan branch.
  2. Adds all project files.
  3. Creates a fresh commit.
  4. Deletes the previous main branch.
  5. Renames the orphan branch to main.
  6. Runs Git garbage collection.
  7. Force pushes the repository to GitHub.

After deployment, the repository contains only one commit representing the current version of the project.

Script

git checkout --orphan current-build
git add --all
git commit -m "Build: $(date)"
git branch -D main
git branch -m main
git reflog expire --expire=now --all
git gc --prune=now --aggressive
git push --force origin main

Advantages

  • Repository size remains almost constant.
  • Faster cloning.
  • Faster GitHub Pages deployments.
  • No accumulation of thousands of generated files.
  • Ideal for generated websites, PWAs, and static documentation.

Disadvantages

  • Complete Git history is removed.
  • Previous commits cannot be restored after the force push.
  • Not appropriate for collaborative projects or repositories where commit history is important.

Best use cases

This approach is useful for repositories containing generated artifacts such as:

  • Eleventy websites
  • Hugo sites
  • Jekyll blogs
  • Vue, React, Angular, or Svelte build outputs
  • PWAs
  • Documentation websites

Recommendation

Keep your source code in a separate repository if preserving development history is important. Use this approach only for repositories where the latest published version is all that matters.

Polished script

#!/usr/bin/env bash

set -Eeuo pipefail

BRANCH="main"
TEMP_BRANCH="current-build"

echo "======================================"
echo " GitHub Pages Deployment"
echo "======================================"

# Make sure we are inside a git repository
git rev-parse --is-inside-work-tree >/dev/null

# Check for uncommitted changes
if [[ -n "$(git status --porcelain)" ]]; then
    echo "Error: Working tree contains uncommitted changes."
    echo "Commit or stash them before running this script."
    exit 1
fi

echo "Removing previous temporary branch (if exists)..."
git branch -D "$TEMP_BRANCH" 2>/dev/null || true

echo "Creating orphan branch..."
git checkout --orphan "$TEMP_BRANCH"

echo "Adding files..."
git add --all

echo "Creating commit..."
git commit -m "Build: $(date '+%Y-%m-%d %H:%M:%S')"

echo "Removing previous $BRANCH branch..."
git branch -D "$BRANCH" 2>/dev/null || true

echo "Renaming branch..."
git branch -m "$BRANCH"

echo "Cleaning repository..."
git reflog expire --expire=now --all
git gc --prune=now --aggressive

echo
echo "Current history:"
git log --graph --oneline --decorate -5

echo
echo "Force pushing..."
git push --force origin "$BRANCH"

echo
echo "Done!"
echo "Repository history now contains a single commit."
