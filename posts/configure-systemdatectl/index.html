Configure System Clock Synchronization with timedatectl

02 Jul 2023 AlexS
bash linux DevOps

Accurate system time is critical for logging, scheduled tasks, certificate validation, and distributed system consistency. A drifting clock can cause subtle bugs that are notoriously difficult to debug — from failed SSL handshakes to corrupted log timestamps.

On modern Linux systems (systemd-based), the timedatectl command provides a simple interface for managing system time and NTP synchronization.

Why Enable NTP Sync?

  • Log accuracy — every log entry carries a correct timestamp across all servers
  • SSL/TLS validity — certificates are time-sensitive; drift causes handshake failures
  • Cron reliability — scheduled jobs fire at the intended time
  • Distributed consistency — microservices and databases rely on synchronized clocks

Setup

First, check the current time status of your system:

timedatectl status

The output shows whether NTP synchronization is active:

      Local time: Mon 2024-09-07 14:30:00 UTC
  Universal time: Mon 2024-09-07 14:30:00 UTC
        RTC time: Mon 2024-09-07 14:30:00
       Time zone: UTC (UTC, +0000)
 System clock synchronized: yes
systemd-timesyncd enabled: yes

If System clock synchronized shows no, enable NTP synchronization:

sudo timedatectl set-ntp true

Verify the change took effect:

timedatectl status

Understanding the Output

FieldMeaning
Local timeCurrent system time in your configured timezone
RTC timeHardware clock (real-time clock) time
Time zoneActive timezone setting
System clock synchronizedWhether NTP sync is currently active and accurate
systemd-timesyncd enabledWhether the systemd time synchronization daemon is running

How It Works

  1. Query status — timedatectl status shows whether your clock is synced with an NTP server
  2. Enable NTP — set-ntp true activates systemd-timesyncd, which automatically contacts upstream NTP servers
  3. Automatic correction — the daemon continuously adjusts the system clock to compensate for drift

Customization Tips

  • Change timezone — run sudo timedatectl set-timezone America/New_York (list options with timedatectl list-timezones)
  • Manual time override — temporarily disable NTP (set-ntp false) and set exact time with sudo timedatectl set-time "2024-09-07 14:30:00"
  • Hardware clock sync — ensure RTC uses UTC with sudo timedatectl set-local-rtc 0 (recommended for most Linux systems)
  • Custom NTP servers — edit /etc/systemd/timesyncd.conf to specify your own NTP pool servers

Two commands that keep your servers in perfect time. Enable NTP sync and never worry about clock drift again.
