Auto-Format Java Sources with Gradle

06 Sep 2024 AlexS
java gradle

Keeping consistent code formatting across a Java project is essential for maintainability and team collaboration. Manually enforcing style guidelines is error-prone and time-consuming. Fortunately, the Spotless Gradle plugin makes it trivial to auto-format your Java sources on every build.

This guide shows you how to integrate automatic Java formatting into your Gradle build using Google's standard code style.

Why Auto-Format?

  • Consistency — every file follows the same style guidelines without manual review
  • Less merge conflicts — unified formatting reduces noise in code reviews and diffs
  • Zero enforcement overhead — formatting happens automatically during compilation
  • Team friendly — new contributors inherit the project's style on first build

Setup

Add the Spotless plugin configuration to your build.gradle file:

plugins {
    // ... other plugins ...
    id 'com.diffplug.spotless' version '6.25.0'
}

spotless {
    java {
        target fileTree('.') {
            include '**/*.java'
            exclude '**/build/**', '**/build-*/**'
        }
        toggleOffOn()
        googleJavaFormat()
        removeUnusedImports()
        trimTrailingWhitespace()
        endWithNewline()
    }
}

// Auto-format on every compile
compileJava.dependsOn 'spotlessApply'

Configuration Breakdown

SettingWhat It Does
target fileTree('.')Scans all .java files in the project directory recursively
exclude '**/build/**'Skips generated build artifacts to avoid unnecessary formatting
toggleOffOn()Enables // OFF / // ON comments for blocks you want to exclude
googleJavaFormat()Applies Google Java Style Guide formatting
removeUnusedImports()Automatically cleans up unused import statements
trimTrailingWhitespace()Removes trailing spaces on every line
endWithNewline()Ensures every file ends with a single newline character
compileJava.dependsOnHooks formatting to run before every compilation

How It Works

Once configured, the formatting runs automatically as part of your normal build workflow:

./gradlew build

The spotlessApply task executes before compileJava, reformatting all source files. You can also run it manually:

# Format all sources
./gradlew spotlessApply

# Check formatting without modifying files
./gradlew spotlessCheck

Customization Tips

  • Different style guide — replace googleJavaFormat() with palantirJavaFormat() or aospJavaFormat() for alternative styles
  • Custom rules — add your own formatting steps with replaceRegex('pattern', 'replacement')
  • CI enforcement — run spotlessCheck in your pipeline to fail builds with improperly formatted code
  • IDE integration — most IDEs can hook into Gradle tasks on file save for real-time formatting

Clean, consistent code without the manual effort. Add Spotless to your build and never argue about code style again.
