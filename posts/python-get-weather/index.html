Python Fetch Weather Data and Render It on an Image

11 Feb 2021 AlexS
python linux raspberryPi

A wall-mounted Raspberry Pi display is perfect for showing real-time weather alongside system stats. The OpenWeatherMap REST API returns current temperature and feels-like data in JSON, and Pillow bakes the values as rotated text directly onto your dashboard image — no browser tab required.

This guide shows how to fetch live weather by city name and draw it vertically on an existing image canvas.

Why Display Weather on a Pi Dashboard?

  • Ambient awareness — see outside conditions without checking a phone or browser
  • Always-on display — wall-mounted screens provide passive weather monitoring
  • Simple REST integration — OpenWeatherMap offers a free tier with city-based lookups
  • Image-native rendering — Pillow draws text at the pixel level, keeping layout consistent

The Solution

import requests
from PIL import Image, ImageDraw, ImageFont

# Fetch weather data from OpenWeatherMap API
api_url = 'http://api.openweathermap.org/data/2.5/weather'
params = {
    'appid': '<openweather_key_id>',
    'q': '<city_name>',
    'units': 'metric'
}
weather_json = requests.get(api_url, params=params).json()

if weather_json["cod"] != "404":
    # Format temperature string: T:actual(feels_like)°C
    degree_sign = u"\N{DEGREE SIGN}"
    temp_actual = int(weather_json["main"]["temp"])
    temp_feels = int(weather_json["main"]["feels_like"])
    text = f"T:{temp_actual}({temp_feels}){degree_sign}C"

    # Load font and render text to grayscale image
    font = ImageFont.truetype('./freefont/FreeMonoBold.ttf', 38)
    image_width = font.getsize(text)
    img_txt = Image.new('L', image_width, 255)
    draw_txt = ImageDraw.Draw(img_txt)
    draw_txt.text((0, 0), text, font=font, fill=0)

    # Rotate 90° for vertical display and paste centered vertically
    t = img_txt.rotate(90, expand=1)
    image.paste(t, (130, int(260/2) - int(image_width[0]/2)))

Replace <openweather_key_id> with your API key from OpenWeatherMap and <city_name> with your target city (e.g., Kyiv or London,GB).

Required Dependencies

Install the required packages:

pip3 install requests Pillow

Make sure you have a TrueType font file (e.g., FreeMonoBold.ttf) in your project directory.

Understanding the Code

ComponentPurpose
requests.get(api_url, params=params)Sends GET request to OpenWeatherMap with query parameters
'units': 'metric'Returns temperature in Celsius (use 'imperial' for Fahrenheit)
weather_json["cod"] != "404"Checks API response — "404" means city not found, "200" means success
weather_json["main"]["temp"]Current air temperature in degrees Celsius
weather_json["main"]["feels_like"]Perceived temperature accounting for wind and humidity
u"\N{DEGREE SIGN}"Unicode degree symbol (°) for proper formatting
ImageFont.truetype()Loads a TrueType font at the specified size (38pt for weather)
font.getsize(text)Returns (width, height) tuple for canvas sizing and centering
Image.new('L', ...)Creates an 8-bit grayscale image sized exactly to fit the text

image.paste(t, (130, y))Pastes rotated weather label at X=130, vertically centered at Y position

How It Works

  1. API request — requests.get() calls OpenWeatherMap with city name and metric units
  2. Response validation — checks "cod" field to confirm the city was found (not 404)
  3. Data extraction — pulls temp and feels_like from the "main" object, rounds to integers
  4. Render text — creates a grayscale image sized to the string, draws black temperature text on white background
  5. Rotate and center — turns label 90° for vertical sidebar placement, centers it at Y = (260/2) - (width/2)

Use the One Call API for multi-hour predictions:

url = f'https://api.openweathermap.org/data/2.5/onecall'
params = {'lat': lat, 'lon': lon, 'units': 'metric', 'appid': '<key>'}
forecast = requests.get(url, params=params).json()

for hour in forecast['hourly'][:3]:  # Next 3 hours

Customization Tips

  • Modern font sizing — replace font.getsize() with font.getbbox(text) for Pillow ≥10 compatibility (getsize is deprecated)
  • GeoIP auto-detect — use the One Call API with IP-based location instead of hardcoding city names
  • Caching — store results in a temp file and only re-fetch every 10 minutes to stay within free tier rate limits
  • Imperial units — switch 'units': 'imperial' for Fahrenheit, or omit the parameter for Kelvin
