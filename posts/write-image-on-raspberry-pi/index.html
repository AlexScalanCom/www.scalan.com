Write Raspberry Pi Image to SD Card via Command Line with dd

13 Jul 2023 AlexS
bash linux DevOps raspberryPi

Flashing a Raspberry Pi OS image to an SD card doesn't require balenaEtcher or Rufus. A single dd command piped from xzcat writes the compressed .img.xz file directly to the block device — fast, reliable, and fully scriptable. This is especially useful on headless Linux machines or remote servers where GUI tools aren't available.

This guide shows how to identify your SD card device, decompress the image on-the-fly, and write it with proper sync to ensure data integrity.

Why Flash from Command Line?

  • No GUI needed — works over SSH on headless servers and remote machines
  • Scriptable automation — integrate into provisioning pipelines for bulk SD card preparation
  • Transparent progress — pipe through pv to see real-time transfer speed and ETA
  • Zero bloat — dd is pre-installed on every Linux distribution, no extra downloads required

The Solution

Find Your SD Card Device

Before writing, identify the correct block device. An incorrect of= parameter will destroy data on the wrong disk:

lsblk
# or
sudo fdisk -l

Look for a removable device (typically /dev/sdb or /dev/mmcblk0) with the expected SD card capacity. Note the exact path — you'll need it in the next step.

Write the Image

Replace <ImageFile> with your downloaded image filename and /dev/sdb with your actual SD card device:

xzcat ./<ImageFile>.img.xz | sudo dd bs=4M of=/dev/sdb conv=fsync

For example, with a Bullseye ARM64 image:

xzcat ./2023-05-03-raspios-bullseye-arm64.img.xz | sudo dd bs=4M of=/dev/sdb conv=fsync

Verify the Write (Optional but Recommended)

After dd completes, confirm data integrity by comparing checksums:

sudo dd if=/dev/sdb bs=4M | sha256sum
sha256sum ./<ImageFile>.img.xz

Understanding the Command

ComponentPurpose
xzcatDecompresses .xz files to stdout without creating a temporary .img file on disk
|Pipes the decompressed image stream directly into dd as input
sudo ddRuns dd with root privileges required for raw block device access
bs=4MSets block size to 4 megabytes — larger blocks mean faster transfers
of=/dev/sdbOutput file is the SD card block device (NOT a partition like /dev/sdb1)
conv=fsyncPerforms physical write of data before finishing, ensuring all buffers are flushed to disk

How It Works

  1. Identify device — lsblk or fdisk -l lists block devices so you can find the SD card's path
  2. Decompress on-the-fly — xzcat reads the compressed .img.xz and streams raw bytes to stdout (no intermediate file needed)
  3. Pipe to dd — the decompressed image data flows into dd as stdin
  4. Write in chunks — dd writes 4MB blocks from stdin to the SD card block device (of=/dev/sdb)
  5. Flush buffers — conv=fsync forces the kernel to sync all pending writes before dd exits, preventing corrupted images

Practical Examples

Show Progress with pv

Install and pipe through pv for a real-time progress bar:

sudo apt install pv
xzcat ./2023-05-03-raspios-bullseye-arm64.img.xz | pv | sudo dd bs=4M of=/dev/sdb conv=fsync

Output: 3.71GiB [========> ] 67% ETA 00:02

Write an Uncompressed .img File

If your image is not .xz compressed, skip xzcat:

sudo dd if=./2023-05-03-raspios-bullseye-arm64.img bs=4M of=/dev/sdb conv=fsync

Write from a .zip Image

unzip -p ./2023-05-03-raspios-bullseye-arm64.zip | sudo dd bs=4M of=/dev/sdb conv=fsync

Safely Eject After Writing

Ensure the SD card is clean before removing it:

sync
sudo eject /dev/sdb

Customization Tips

  • Use status=progress — add status=progress to the dd command for built-in progress output (GNU coreutils ≥ 8.24): dd bs=4M of=/dev/sdb conv=fsync status=progress
  • Truncate before write — for reuse on the same card, zero the first few sectors: sudo dd if=/dev/zero of=/dev/sdb bs=4M count=10
  • Enable SSH on first boot — create an empty ssh file on the boot partition after flashing: touch /media/$USER/boot/ssh
  • Preconfigure WiFi — add a wpa_supplicant.conf file to the boot partition before first boot

One decompress stream, one dd command. The image lands on your SD card, synced and ready to boot.
