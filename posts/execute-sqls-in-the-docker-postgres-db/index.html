Execute SQL Commands in a Docker PostgreSQL Database

12 Jul 2022 AlexS
bash docker postgres

Working with PostgreSQL inside Docker means you can't simply open psql locally — you need to pipe commands into the running container. The docker exec combined with a heredoc or file redirect makes it trivial to run inline SQL statements or import entire dump files without ever entering the container shell.

This guide shows how to execute SQL commands and restore database dumps against a Dockerized PostgreSQL instance.

Why Use docker exec for SQL?

  • No shell access needed — run queries without attaching an interactive psql session
  • Script-friendly — inline SQL works seamlessly in CI/CD pipelines and provisioning scripts
  • Database lifecycle — quickly create, drop, or reset databases during local development
  • Dump imports — restore full database backups with a single command

Execute Inline SQL Commands

Use a heredoc to send multiple SQL statements to the container in one shot:

docker exec -i <db_container_name> psql -U <username> <<-EOSQL
  DROP DATABASE IF EXISTS dbname;
  CREATE DATABASE dbname;
EOSQL

Replace <db_container_name> with your actual PostgreSQL container name (e.g., my-postgres) and <username> with the database user (typically postgres).

Run a Single Query

For quick one-liners, skip the heredoc:

docker exec -i my-postgres psql -U postgres -d dbname -c "SELECT count(*) FROM users;"

Import a SQL Dump File

Restore an entire database from a .sql dump by redirecting the file into psql:

docker exec -i <db_container_name> psql -U <username> -d dbname < dumpfile.sql

This is the fastest way to seed a fresh database or migrate data between environments.

Understanding the Flags

FlagPurpose
docker exec -iExecutes a command in a running container with stdin kept open (required for heredoc and file redirect)
<db_container_name>Name or ID of the PostgreSQL Docker container
psql -U <username>Invokes the PostgreSQL CLI as the specified user
-d dbnameConnects to a specific database (optional, defaults to user's default database)
-c "query"Runs a single SQL command and exits
<<-EOSQL ... EOSQLBash heredoc that feeds multiple SQL lines into psql
< dumpfile.sqlShell redirect that pipes the contents of a SQL file into psql

How It Works

  1. docker exec — launches psql inside the running PostgreSQL container, forwarding stdin
  2. Heredoc or redirect — feeds raw SQL statements through stdin into the psql process
  3. psql executes — processes each SQL statement sequentially and writes results to stdout
  4. Container stays running — the command exits after execution but the container continues uninterrupted

Practical Examples

Reset a Development Database

docker exec -i my-postgres psql -U postgres <<-EOSQL
  DROP DATABASE IF EXISTS myapp_dev;
  CREATE DATABASE myapp_dev;
  \c myapp_dev
  CREATE TABLE users (id SERIAL PRIMARY KEY, email VARCHAR(255) NOT NULL);
EOSQL

Import a Production Dump Locally

# First create the target database
docker exec -i my-postgres psql -U postgres -c "CREATE DATABASE myapp;"

# Then import the dump
docker exec -i my-postgres psql -U postgres -d myapp < production_dump.sql

Export a Database from Docker

The reverse operation — extracting a dump from the container:

docker exec -i my-postgres pg_dump -U postgres myapp > local_backup.sql

Customization Tips

  • Specify host and port — add -h localhost -p 5432 if psql needs explicit connection parameters
  • Suppress output — pipe stdout to /dev/null for silent execution in scripts: docker exec ... > /dev/null
  • Set password via env — use PGPASSWORD=mysecret docker exec ... when authentication requires a password
  • Use with docker-compose — reference services by name: docker-compose exec -T postgres psql -U postgres -c "..."
