Configure .htaccess for React App

16 Dec 2024 AlexS
apache nginx DevOps

Problem: Deploying a React Single Page Application (SPA) in a subdirectory on an Apache server results in 404 errors when users refresh the page or navigate directly to a routed path — because the requested URL doesn't correspond to a physical file on disk.

Why This Happens

React Router handles routing entirely on the client side. When you navigate to /app/dashboard, the browser requests /subdir/app/dashboard from the server. Apache looks for a real file at that path, finds nothing, and returns a 404 Not Found. The solution is to rewrite all non-file requests back to index.html, letting React take over routing.

Solution — .htaccess Configuration

Place the following .htaccess file in the same directory where your React build output (index.html, static assets) lives on the server:

# Disable MultiViews to prevent unwanted content negotiation
Options -MultiViews

# Enable the rewrite engine
RewriteEngine On

# Preserve Authorization header (required for protected APIs behind Apache)
RewriteCond %{HTTP:Authorization} ^(.*)
RewriteRule .* - [e=HTTP_AUTHORIZATION:%1]

# If the request is NOT for an existing file, route it to index.html
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.html [E=X-HTTP_AUTHORIZATION:%{HTTP:Authorization},QSA,L]

How It Works — Line by Line

DirectivePurpose
Options -MultiViewsDisables Apache content negotiation which can interfere with SPA routing.
RewriteEngine OnActivates mod_rewrite for URL rewriting rules.
RewriteCond %{HTTP:Authorization} ...Captures the Authorization request header so it doesn't get stripped during internal rewrites (critical for authenticated apps).
RewriteRule .* - [e=HTTP_AUTHORIZATION:%1]Stores the captured header as an environment variable.
RewriteCond %{REQUEST_FILENAME} !-fOnly applies the rewrite when the requested path does not match an existing physical file (CSS, JS images bypass this rule).
RewriteRule ^ index.html [...]Redirects everything else to index.html, passing along query string args (QSA), forwarding the auth header, and marking as last rule (L).

Directory Structure Example

/var/www/html/
├── index.html            ← Your main site
└── app/                  ← React SPA subdirectory
    ├── .htaccess         ← The config above
    ├── index.html        ← React build entry point
    ├── static/
    │   ├── css/
    │   └── js/
    └── asset-manifest.json

In this setup, any request like example.com/app/dashboard or example.com/app/settings/profile will correctly serve the React index.html, and client-side routing will handle the rest.

Quick Verification Checklist

  • [ ] mod_rewrite is enabled on your Apache server (a2enmod rewrite).
  • [ ] Your <Directory> block allows .htaccess overrides (AllowOverride All).
  • [ ] Static assets (CSS, JS, images) load correctly and are not being rewritten.
  • [ ] Direct navigation and page refresh on any route work without 404 errors.

Nginx Equivalent

If you're running Nginx instead of Apache, the equivalent configuration in your server block is:

location /app/ {
    try_files $uri $uri/ /app/index.html;
}

This straightforward rewrite strategy ensures your React SPA routing works seamlessly behind Apache — no build-time path hacks or framework-specific plugins required.
