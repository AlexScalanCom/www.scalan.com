Monitor AWS EC2 Instance Status from a Bash Script

18 Jan 2021 AlexS
bash linux DevOps

AWS CloudWatch handles monitoring at the platform level, but sometimes you need lightweight, ad-hoc health checks from a terminal — checking whether an EC2 instance passed its system and instance status checks without opening the console. The AWS CLI exposes the describe-instance-status command that returns real-time health data queryable with JMESPath.

This guide shows how to build a bash script that monitors both instance-level and system-level status for any EC2 instance.

Why Monitor EC2 Status via Bash?

  • Quick health checks — verify instance status without logging into the AWS console
  • Automation-friendly — integrate status checks into cron jobs, deployment scripts, or alerting pipelines
  • Two-dimensional health — separate instance checks (OS-level: CPU load, network) from system checks (hardware, hypervisor)
  • Zero dependencies — only requires the AWS CLI, no third-party monitoring tools

The Solution

Create a script that queries both instance and system status for a given EC2 instance ID:

#!/bin/bash

instanceID="i-0abc123def456"  # replace with your instance ID

# Get instance status (OS-level health: CPU, network, etc.)
instanceStatus=$(aws ec2 describe-instance-status \
  --region us-east-1 \
  --instance-ids "$instanceID" \
  --query 'InstanceStatuses[0].InstanceStatus.Details[0].Status' \
  --output text)
echo "Instance status: $instanceStatus"

if [ "$instanceStatus" = "ok" ]; then
    echo "Instance check status passed..."
fi

# Get system status (hardware-level health: hypervisor, host, etc.)
systemStatus=$(aws ec2 describe-instance-status \
  --region us-east-1 \
  --instance-ids "$instanceID" \
  --query 'InstanceStatuses[0].SystemStatus.Details[0].Status' \
  --output text)
echo "System status: $systemStatus"

if [ "$systemStatus" = "ok" ]; then
    echo "System check status passed..."
fi

Replace "i-0abc123def456" with your actual instance ID and adjust the --region to match your deployment.

Understanding the Checks

Check TypeWhat It MonitorsPossible Values
Instance StatusOS-level health: CPU credit balance, network I/O, SSH connectivity, disk I/Ook, impaired, not-applicable
System StatusHardware-level health: host system, hypervisor, underlying infrastructureok, impaired, not-applicable

Note: The status values returned are ok and impaired (not passed as shown in older AWS CLI documentation). Use "ok" for reliable comparisons.

Understanding the Command

ComponentPurpose
aws ec2 describe-instance-statusAPI call that returns health check results for specified instances
--region us-east-1Targets the AWS region where your instance runs
--instance-ids "$instanceID"Specifies which instance(s) to check
--query 'InstanceStatuses[0].InstanceStatus.Details[0].Status'JMESPath expression extracting the instance-level status string
--query 'InstanceStatuses[0].SystemStatus.Details[0].Status'JMESPath expression extracting the system-level status string
--output textReturns plain text instead of JSON for easy shell comparison

How It Works

  1. Instance query — first aws ec2 describe-instance-status call extracts OS-level health via JMESPath into $instanceStatus
  2. Instance check — compares the result against "ok" and logs a pass message if healthy
  3. System query — second call with a different JMESPath path extracts hardware-level health into $systemStatus
  4. System check — compares system status against "ok" and logs a pass message if infrastructure is healthy

Practical Examples

Monitor Multiple Instances

for id in i-0abc123 i-0def456 i-0ghi789; do
  status=$(aws ec2 describe-instance-status \
    --region us-east-1 --instance-ids "$id" \
    --query 'InstanceStatuses[0].InstanceStatus.Details[0].Status' \
    --output text)
  echo "$id : $status"
done

Alert on Impaired Status

instanceID="i-0abc123def456"
status=$(aws ec2 describe-instance-status \
  --region us-east-1 --instance-ids "$instanceID" \
  --query 'InstanceStatuses[0].InstanceStatus.Details[0].Status' \
  --output text)

if [ "$status" != "ok" ]; then
  echo "ALERT: Instance $instanceID health is $status" | mail -s "EC2 Alert" admin@example.com
fi

Continuous Monitoring with Cron

Add to crontab (crontab -e) for checks every 5 minutes:

*/5 * * * * /path/to/monitor-ec2.sh >> /var/log/ec2-health.log 2>&1

Customization Tips

  • Dynamic instance lookup — combine with aws ec2 describe-instances --filters to resolve instance IDs by tag name instead of hardcoding
  • Include cause details — extend the JMESPath query to also grab .Cause for diagnostic messages
  • JSON output for logging — switch to --output json and pipe through jq for structured log entries
  • Multi-region support — loop over an array of regions if your instances span multiple AWS regions

Two API calls that give you full visibility into EC2 health. Check the OS, check the hardware, automate the alerts.
