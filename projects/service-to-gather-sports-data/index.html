Service to Gather Sports Data

  • 28 Jun 2022
java springboot

Project duration: 2 years · Domain: Sports Analytics / Data Aggregation / Java Backend

Overview

Designed and developed a robust data-aggregation microservice responsible for collecting, normalizing, and distributing sports analytics data across three major American leagues (MLB, NFL, NBA). The platform ingests raw feeds from four external providers, enriches and harmonizes the datasets, persists everything to a centralized MySQL store, and serves consumers via a cache-accelerated REST API — all hosted on AWS infrastructure.

Responsibilities & Achievements

  • Multi-source data ingestion — Engineered scheduled harvesters (powered by cron-driven Spring Boot tasks) that pull data from SportRadar, Stats, FantasyPros, and OpenWeather APIs; implemented retry logic, rate-limit handling, and payload validation to guarantee feed reliability across every execution cycle (every 10 minutes, twice daily, and weekly jobs).

  • Data normalization & enrichment — Mapped heterogeneous schemas from each provider into a unified internal data model covering player maps, consensus rankings, team rosters, depth charts, season projections, seasonal game projections, season statistics, game rosters, weekly league leaders, team hierarchies, season schedules, weekly schedules, rankings, standings, and game summaries — plus correlated weather data for every event location.

  • REST API development — Designed and documented a comprehensive suite of RESTful endpoints enabling client sites to query any aggregated dataset; applied consistent pagination, filtering, and response-wrapping conventions for seamless consumer integration.

  • Performance optimization with caching — Integrated Memcached as a distributed caching layer to intercept and serve high-frequency read requests, dramatically reducing database load and cutting average API response times.

  • AWS infrastructure deployment — Provisioned and maintained the application on an AWS EC2 instance backed by Amazon RDS (MySQL) and a dedicated Memcached node; configured security groups, automated backups, and health-check monitoring for continuous uptime.

  • CI/CD & version control — Managed the full development lifecycle using Maven for builds, Git with GitFlow branching strategy for structured releases, and automated deployment scripts to promote code from feature branches through staging to production.

Delivered a single source of truth for cross-league sports data, eliminating redundant integrations for downstream client applications. The cron-based ETL pipeline processed millions of records daily with Memcached-driven sub-second API responses, powering multiple consumer platforms throughout the two-year engagement.
