Set of REST API Services

  • 15 Mar 2023
webservice java springboot

Project duration: 2 years · Domain: Backend Architecture / REST API Platform / Online Design Tool

Overview

Architected and developed a comprehensive Java-based backend platform from the ground up to support an online design tool. The system comprises 50+ production-grade REST API endpoints, robust authentication & authorization, PostgreSQL persistence with Liquibase-driven migrations, and cache-layer optimization — delivering reliable, low-latency data services consumed by multiple client-facing web applications over a two-year engagement.

Responsibilities & Achievements

  • REST API design & implementation — Designed, developed, and maintained a suite of 50+ RESTful endpoints following consistent resource naming, HTTP semantics, pagination, filtering, and error-handling conventions; each endpoint enables client applications to create, read, update, and delete design-related entities stored in PostgreSQL.

  • Authentication & authorization — Integrated secure token-based authentication (JWT/OAuth-inspired flow) across all endpoints, ensuring role-aware access control so only authorized users and services could interact with protected resources.

  • Database schema management — Leveraged Liquibase for version-controlled database migrations, including initial schema provisioning, incremental evolution, seed data initialization, and seamless rollback strategies — guaranteeing zero-downtime schema deployments across environments.

  • Data persistence with Hibernate/JPA — Built a clean, layered domain model using Hibernate ORM to map complex entity relationships; implemented optimized JPQL queries, lazy/eager loading strategies, and transaction boundaries to maximize throughput.

  • Performance optimization — Applied response caching for high-frequency read-heavy endpoints, significantly reducing database round-trips and improving API latency; conducted load testing with Apache jMeter to identify bottlenecks and validate performance SLAs under peak concurrent-user scenarios.

  • Build & delivery pipeline — Utilized Gradle for modular multi-project builds; enforced GitFlow branching strategy for structured release cycles; tracked epics, stories, and defects via Jira ensuring full transparency from backlog to production deployment.
