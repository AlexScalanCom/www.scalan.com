Support Java Set of Services

  • 31 Aug 2023
fintech DevOps

Project duration: 6 months · Domain: Fintech / DevOps Operations

Overview

Served as the primary DevOps engineer responsible for maintaining and optimizing the deployment pipeline of a Java-based microservices ecosystem running on a commercial bank's production infrastructure. Ensured seamless, compliant release cycles while bridging communication between development, QA, and release management teams across a highly regulated financial environment.

Responsibilities & Achievements

  • Release coordination — Acted as the key liaison with the release management team to scope, prioritize, and track every incoming update; maintained full visibility into what changes were deployed where, ensuring strict change-management compliance.

  • Change request lifecycle — Prepared, validated, and executed formal update requests through the bank's approval workflow; verified artifact integrity and rollback readiness before each deployment window.

  • Post-deployment validation — Monitored deployment status in real time, analyzing application and system logs to confirm successful releases; rapidly identified and escalated anomalies to minimize production risk.

  • Cross-team collaboration — Conducted regular syncs with QA and development teams to refine deployment procedures, address pipeline bottlenecks, and improve overall release velocity without sacrificing stability.

  • CI/CD script modernization — Authored and maintained Bash automation scripts for CI/CD pipelines (TeamCity and uDeploy), introducing error handling, logging improvements, and reusable modules to reduce manual intervention and human error.

  • Production server monitoring — Implemented proactive health checks and resource monitoring across Red Hat Enterprise Linux servers; leveraged SSH and native Linux diagnostics to track JVM performance, disk I/O, network latency, and service uptime.
